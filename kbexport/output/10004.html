Error running GRITS report

10004

Introduction

This article addresses a bug in EproLib4 Version 4.05.33.3 (EproLib4.dll Version 1.6.0.0) that causes the GRITS report to fail.

Symptom

  • The GRITS report will not run
  • The EncounterPRO Report service hangs
  • The following error can be seen in the Event Log when the GRITS report fails:
    Error calling external object function printreport at line 35 in function xx_printreport of object u_component_report.

Cause

EproLib4 Version 4.05.33.3 (EproLib4.dll Version 1.6.0.0) has a bug that causes a subscript out of range error in the EproLib4.GRITS.PrintReport function.

Workaround

Resolution

This bug is fixed in EproLib4 Version 4.05.33.4 (EproLib4.dll Version 1.7.0.0). It can be found on the FTP Site under Support Installers.

Notes

Installation of EproLib4 will require a restart if EproLib4.dll is in use at the time of installation.  To avoid having to restart, close all instances of EncounterPRO, and stop all EncounterPRO services before installing or upgrading EproLib4.
