One Workstation PC cannot print Word report

10043

Introduction

This article addresses a problem where a particular workstation cannot view or print Word reports

Symptom

Clicking on the word report for example “GA Immune” the user does not get an option to select printer or preview report.

In Event Viewer:
>>> u_common_thread - (set_printer()) Error setting default printer in registry (MS Document Image Writer)

Cause

The most common reason why the workstation does not get these optionsis because Microsoft Word is not installed on this workstation.

Workaround

A workaround to allow a workstation that does not have Microsoft Word installed is to set up the report to have a default printer when “Print from Server” is selected. To set this up the following needs to be done

Go to c_report_definition

Search for report name under the “description“ column. For example “Ga Immune“

Copy report ID. Example “{B1261FE1-22C9-19E9-47F2-D4D6F0EB8276}“

Go to o_report_printer and search for this report ID. If there is no matching records create the below record. Usually you will only have to change the printer name. If there are multiple offices you will have to create a seperate row per office.

Restart “EncounterPRO Report“

Close EncounterPRO and re-open the application and print from server should now work

dbo_o_Report_Printer
