Well Visit going to No Services

10052

Introduction

Symptom

After completing an encounter service (e.g. the Allergy Screen) the patient shows the “No Services“ bar instead of the next scheduled service.  The event viewer shows the following error:

Maximum stored procedure, function, trigger, or view nesting level exceeded (limit 32)

Cause

This is caused by too many items in a single workplan step that are skipped.

Workaround

There is no workaround.  There is a permanent fix in Hotfix 20 and in database modification level 121.

Resolution

Run Hotfix 20 or upgrade the database to Modification Level 121.
