Error Message Screen results were not saved

10054

Introduction

The error message:  "Screen results were not saved" appears in the event viewer

Symptom

The error message:  "Screen results were not saved" appears in the event viewer when the user exits the test results edit/review screen.

Cause

When the user exits the test results screen either by clicking “Done“ (edit mode) or “OK“ (review mode), EncounterPRO counts the number of results on the screen and then counts the number of results in the database.  If the numbers don't match then EncounterPRO assumes that some screen edits have not been persisted to the database and it logs this error.

 

 

 

Workaround

If the user is in “Edit“ mode (where there is a “Done“ box with four options), then the user will get a popup error message on the screen which reads "You cannot complete this lab/test without entering results".  When this happens the user should restart EncounterPRO and try to enter the results again.  If the problem still happens then the user should contact JMJ Customer Support.

 

If the user is in “Review“ mode (only an “OK“ button at the bottom right), then they will not get an popup error message because EncounterPRO doesn't need to update the database.  In this case the error message in the event viewer is not a problem.
