Introduction

Error calling external object function ...

Symptom

Sometimes an operation in EncounterPRO fails with an error message that contains the phrase “Error calling external object function ”.  The may vary from instance to instance.

Cause

EncounterPRO uses many different pieces of software which are incorporated in the form of COM objects.  These COM objects must be registered in the windows registry in order for EncounterPRO to make use of them.  Usually, the EncounterPRO installation program takes care of installing and registering the COM objects.  Occaisionally, a COM object might get corrupted or unregistered.  When this happens, the next time EncounterPRO tries to use that COM object, the error “Error calling external object function ” will be reported to the event viewer.

Workaround

Resolution

The first thing to look for is a JMJ knowlege base article for the specific object function mentioned in the error message.  If there is one, follow its intructions.  If other knowlege base articles don't exist or don't work, follow these instructions:

Most of the COM objects EncounterPRO uses are stored in two software libraries called EproLib4 and EproLibNet.  Both of these libraries have stand-alone installation programs which can be found in the EP_SupportingInstallers on the FTP site.  Try installing EproLib4 and retest EncounterPRO.  If the problem persists, install EproLibNet and retest.
