Incoming messages are not picked up by transport in

10062

Introduction

This article explains what to do if an  incoming message is not picked up by Transport In

Symptom

Messages are queued up in shared folder but Transport In is not picking them

Cause

1. Permissions to TransportIn service logon user

Most of the times it's with permissions to 'eprojmj' user account. Make sure this account is set with local administrative privileges.

2. after centricity 2004 upgrade

If the practice is upgraded to Centricity 2004 then there could  be a problem with AHCHL7ASCII.DLL. GE sent us the updated DLL and you can find this from FTP under ep_integrations\centricity2004. Copy these DLL's to system32 and re-register them. That should fix it.

3. file type

HL7: Check in HkeyLocalMachine->Software->AHC->Transport->EproSever to see whether the FileExtension is matching up with the incoming file type.

Non-HL7: check o_message_subscription table for the component_id "TRN_IN_FILECOPY" to see whether the file extension is matching up with the incoming file type.

Also check the shared folder to see whether it's having full permissions.
