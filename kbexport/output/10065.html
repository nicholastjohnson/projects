Delete the files saved in Holdbills directory

10065

Introduction

with 33.14 installation there's an exe in encounterpro install folder called “PURGE.EXE“. This accepts 2 arguments and the syntax is below

purge.exe <directory> <arugments>

example: purge.exe “d:\jmj\pmshare\holdbills\*.*” -age 14

the above command will delete files older than 14 days from that directory

Symptom

Cause

Workaround

Resolution

By creating a sql job and scheduled to run every day we can purge all the files older than 14 days from holdbills directory. this saves a lot of disk space for large customers. we have never done as part of our installation before and so many of our customers are flooded with too many files[large customers may have 30K files] under holdbills folder. please create a sql job as per the steps below:

1) Create a sql job; add a new step and select the Type as “Operating System Command(CmdExec)” and type in following code under command

“<encounterpro application path>\purge.exe” “<directory>\*.*” -age 14

the above command deletes all the files older than 14 days.
