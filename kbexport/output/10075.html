Resource appointments are going under wrong provider

10075

Introduction

this article explains why the patients are not assigned to a provider or going under wrong provider.

Symptom

a) known issue with Centricity 2004 and this may not be true for all practices. some practices setup the resource appointments differently from the other and so it's not everyone will experience this after centricity 2004 upgrade

b) the PM resource codes are changed or not mapped in encounterpro.

Cause

1. If  Centricity PM,

 

I am seeing a minor issue with one of our centricity 2004 practice with resource schedules. While scheduling a resource on the schedule screen it shows the resource name on both resource and doctor field. if this is true then there are no probs. But some practices are setup differently.. when that screen comes up it shows the actual resource in resource fields and the doctor field shows the actual provider. If this is the case then with centricity 2004 all the resource appointments will go under whatever filled in DOCTOR field.

 

2. extenders are not mapped in c_user or b_resource table.

 

 find the resource code from PM System and map them in c_user or in b_resource. look in o_message_log.comments field  for that patient# to see what transport in reported while processing that message.
