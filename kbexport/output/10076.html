Error opening window when background color is changed

10076

Introduction

Not all screens in EncounterPRO behave correctly when the background color is changed in preferences.

Symptom

When opening a window such as the Prescription Edit screen, The screen is not filled in and the following error appears in the event viewer:

 

Event Type:       Error

Event Source:    EncounterPRO

Event Category: None

Event ID:           0

Date:                12/9/2004

Time:                12:47:50 PM

User:                N/A

Computer:         DCMED

Description:

EPMD\jmjuser on DCMED

4.05.33.7 (7/27/2004) >>> UNKNOWN CALLER - (SYSTEM ERROR) errornum:2

message :Null object reference at line 37 in open event of object w_window_base.

where   :w_window_base

object  :w_window_base

event   :open

line    :37

Cause

This error is caused when the user has changed the default background color and attempts to display a window which does not support changing the background color.  As of 4.05.33.15, changing the bcakground color is not officially supported.

 

Workaround

Resolution

Remove the preference which changes the background color.  To remove all background color preferences, run the following query against the database:

DELETE
FROM o_Preferences
WHERE preference_id = 'color_background'
GO
