How to automatically set the Primary Care Provider

10079

Introduction

If the primary care provider is not set through the PM interface, the practice may elect to have EncounterPRO automatically set the primary care provider.  This article explains how to configure EncounterPRO to automatically set the Primary Care Provider to the last provider who saw the patient for a Well Visit.

Symptom

 

Cause

 

Workaround

 

Resolution

Have the SQL Server Administrator create a daily SQL Agent job which runs the following SQL script:

------------------------------------------------------------------

EXECUTE sp_update_pcp

------------------------------------------------------------------

Notes and Limitations:

This script will set the Primary Care Provider to the provider who owned the last WELL Visit.  A WELL visit is defined as an encounter where an assessment was created where the assessment type is WELL.

The Primary Care Provider will only be updated if the encounter owner is in the Physician role

The Primary Care Provider will only be updated if the encounter took place during the 30 days prior to the script running

The Primary Care Provider will not be updated if the encounter contains a progress record where the progress_type is 'Property', the progress_key is 'Keep PCP', and the progress_value is 'True'.  To give the user the option to do a well visit and have EncounterPRO not update the PCP, paint a button on the SOAP screen (or anywhere with the encounter context) that logs the appropriate progress record.
