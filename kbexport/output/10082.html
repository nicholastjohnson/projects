Patient encounters going to Nowhere

10082

Introduction


Symptom


Cause

If the workplan does not have a default room or have the wrong default room then it may go to "nowhere"

Workaround


Resolution

1. Go to configuration, encounter types, select the indirect tab and choose the correct encounter type

2. Select your edit encounter type option and look for the name of the workplan associated with the encounter type.

3. Then go to Worplans, change the workplan type from patient toEncounter and search for the workplan that is linked to the encountertype that is going to "nowhere"

4. Once you locate the workplan, select it and select edit workplan

5. If a local copy of the workplan hasn't been made yet you will not abe able to edit the workplan until you do so.

6. Once a local copy has ben made, select the edit workplan button to the right of the screen.

7. Select the first step in the workplan and choose "room Assignment"

8. Change the default type to the name of the practice or site you want to set the room for

9. For telephone calls, select the examination option on the left and your room selection will display on the right.

10. Select the appropriate room this encounter should defualt to.

11. That's it, problem solved
