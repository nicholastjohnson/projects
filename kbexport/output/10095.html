How to Use and Choose Treatment Workplans for a Send-Out Labs

10095

Introduction

"Send-Out Lab" is a new observation and new, easier way to order multiple labs and to generate a referral RX.  This new functionality is available in Configuration Versions 21 or higher. 

Symptom

The problem that needed to be solved was two-fold.  First, if a doctor ordered multiple send-out labs the nurse had to complete the send-out information and click "collected" for each of those labs.  Second, consolidated lab reports were coming back to the doctor and the consolidated report was being scanned into each individual lab.

Cause

Workaround

A workaround of a new treatment_type called "Referred Treatment Items" was attempted in Configuration Beta Version 21; however, this meant that lab history for two identical labs could be stored either as a Lab treatment type or as a Referred Treatment Item treatment type.  This meant that physicians would have to search in two different places for lab history. 

Resolution

A new (optional) lab observation called "Send-Out Lab(s)" was created and new treatment workplans were created for this new observation.  The following treatment workplans were created:

Workplans                                                                                                

1) Send-Out Lab/ Optional Collect by Staff/ Paper Work InOffice, Staff     

2) Send-Out Lab/ Collect by Staff/ Paper Work InOffice, Coord                 

3) Send-Out Lab/ Paper Work InOffice, Coord                                          

4) Send-Out Lab/ Collect by Staff/ Paper Work InOffice, Coord                 

5) Send-Out Lab/ Paper Work InOffice, Coord                                          

 

Only one treatment mode (Referred TX) was put into the Master DB and it will automatically apply to newly-installed offices with ids of 0001 to 0020.  This treatment mode is associated with workplan  1) above.  

   

  Change the workplan associated with the treatment mode if needed.  For now, the reports require either a treatment description starting with the word "Referred" or a treatment mode of 'Referred TX."

For existing installations, the following script needs to be run  (Dev Issue 5323): 

INSERT INTO o_treatment_type_default_mode
(
 treatment_type,
 treatment_key,
 office_id,
 treatment_mode,
 created_by
)

SELECT distinct 
 'LAB',
 '0^24197',
 c.office_id,
 'Referred TX',
 'SUPPORT'

FROM o_treatment_type_default_mode dm
 LEFT OUTER JOIN c_office c
 ON dm.office_id = c.office_id

WHERE c.office_id IS NOT NULL
AND NOT EXISTS  (SELECT treatment_mode
  FROM o_treatment_type_default_mode
  WHERE treatment_mode = 'Referred TX'
  AND office_id = c.office_id)
  

 

Notes

The customer needs to be asked prior to installation a) whether blood is drawn for labs in the office, b) whether the nurse or the front desk or a special referral scheduler does the paper work.
