EncounterPRO does not allow certain users to log in

10096

Introduction

 

Symptom

  1. EPRO does not allow certain users to log into application
  2. EPRO closes after you select the log on button

Cause

This happens when
1) user has themselves as their supervisor
2) is in a supervisory tree with a loop
     example:  user 1 's supervisor is user 2
                    user 2's supervisor is user 3
                    user 3's supervisor is user 4
                    user 4's supervisor is user 3
none of the above users can log in

Workaround

 

Resolution

Remove Supervisor from c_user or from EncounterPRO
