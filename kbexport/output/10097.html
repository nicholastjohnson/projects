Problem with viewing a image in version 4.0

10097

Introduction

Symptom

  1. EncounterPRO closes when you try to view an attachment
  2. You get one of the below errors when viewing an attachment

UNKNOWN CALLER - (f_convert_tiff_to_bmp()) Error converting image file from "c:\temp\jmj_325__041214151434_2.tif" to "c:\temp\jmj_325__041214151434_3.bmp":  Error calling external object function convertimage at line 18 in function f_convert_tiff_to_bmp of object f_convert_tiff_to_bmp.

Ver. NA >>> EPImageViewer - (.ctor) Error Loading Image in file c:\temp\jmj_368_70373402__041221061727_1.tif System.IO.FileLoadException: The located assembly's manifest definition with name 'Atalasoft.dotImage' does not match the assembly reference.
File name: "Atalasoft.dotImage"
   at EPImageViewer.ImageViewer..ctor()
   at EPImageViewer.Image.LoadImageFromFile(String fileName, Boolean readOnly)

 

Cause

Workaround

Resolution

1. Delete the following files from the EncounterPRO program folder:

a. AtalaImaging.dll

b. Atalasoft.dotImage.dll

c. Atalasoft.dotImage.Pdf.dll

d. Atalasoft.dotImage.WinControls.dll

e. EPImageControl.dll

f. EPImageViewer.dll

2. Install the latest EproLibNET (4.05.33.16)  

3. Perform a file search for "regasm.exe, epimageviewer.dll"

4. In the search results window, you should see one copy of EPImageViewer.dll and one or more of RegAsm.exe

5. Drag EPImageViewer.dll onto the latest version of RegAsm.exe (if there is more than one) to manually register the .NET library on the computer

Notes

This is usually only affected with 33.15
