Symptom

  1. Manual post of lab results does not yield "send to-do" service.
  2. Automatic posting of labs does not send to-do item to appropriate physician.
  3. .xml files being posted in patient chart.
  4. Manual post of .xml lab results requests a folder selection.
  5. Practice does not have "Daily" referral report.
  6. Results from two different lab results are posing to a single observation.
  7. Lab result "import" service is timing out.

Cause

Numbers 1 & 2 are related to the default treatment workplan.

Numbers 3  and 4 are related to a missing table entry that directs .xml files to a specific folder

Number 5 is related a missing button that needs to be added

Number 6 is related to an incorrect association that EncounterPRO has established between two different lab result sets to a single lab observation.  (Example: TSH values posting to CBC's when TSH values should post to TSH lab observations only.)

Number 7 is a consequence of not having the "JMJ Lab Results" external source properly identified on the client's database server.

Workaround


Resolution

  • Practices that are utilizing lab integration via .xml file format, need to set the default treatment workplan for labs to: 
    Lab Ongoing Clinical Staff Collect or Send/Refer Out (Integration)
    ALSO NOTE: Wait service must be configured to include:
    Progress Type= Results
    Progress Key= Status

    Also, the Wait service can be configured to include: Progress Type= Results and Progress Key= Status by running this script:

    Insert into c_workplan_item_attribute (workplan_id, item_number, attribute, value)
     Values ('2192','8134','wait_progress_type','results')
    go
    Insert into c_workplan_item_attribute (workplan_id, item_number, attribute, value)
     Values ('2192','8134','wait_progress_key','Status')
    go

  • To resolve the incorrect association of lab results to lab observations, open MS Access and find the dbo_c_XML_Code table.  Delete all records with the code_domain of Labcorp Catalog.  During training it is not unlikely that incorrect associations could be made and therefore it is a good idea to begin with a blank slate before the "live" production environment starts.
  • To properly configure the external sources of the server, log onto the server and open EncounterPRO.  Go to the configuration tab and select "External Sources."  Find the JMJ Lab Results external source and make sure that it is highlighted.
