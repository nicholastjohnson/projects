Prescriptions are not automatically printing

10102

Introduction

DO NOT USE THIS SCRIPT TO Resolve Issues - Please see Support.  This article covers the workaround for when a practice says their prescriptions won't print automatically (but will print manually). 

Symptom


Cause


Workaround

DO NOT USE THIS SCRIPT TO RESOLVE AN ISSUE - Please see Support

NOTE:  This workaround is no longer required with Epro Server version 5.1.9.  If the customer has not yet been upgraded to Epro Server 5.1.9, consider the upgrade at this time.

Version 4.0

  • Open up Query Anazlyer
  • Pause Printers
  • Run Query # 1
  • If there are any returned rows run Query # 2.
  • Restart the EncounterPRO Report service or EncounterPRO Service
  • Go back Query Anazlyer and run Query # 1
  • If there are any results with the status of Starting or Started. Run Query #2
  • Ensure that Query # 1 runs without any returned rows
  • Cancel all Print Jobs in Printer and unpause printers
  • Advise client that problem is resolved &that we have a permament solution that is in currently in Testing

Query # 1

SELECT patient_workplan_item_id, cpr_id, encounter_id, description, dispatch_date, status, retries
FROM p_Patient_WP_Item WITH (NOLOCK)
WHERE owned_by='#JMJ'
AND status not in ('Completed', 'Cancelled', 'Expired')
AND ordered_service IN ('CLOSE_TREAT', 'REPORT')
AND begin_date IS NOT NULL
AND begin_date < DATEADD(minute, -2, getdate())


Query # 2

Run Close Prescriptions for JMJ

Version 2.0

  • Pause Printer
  • Clear all items from o_report_queue
  • Restart EncounterPRO Report Service
  • Cancel all Print Jobs in Printer and unpause printers
  • Follow below KB article

https://www.jmjtech.com/kb/Article.aspx?ArticleNumber=10374

Resolution

Problem is resolved with EPRO SERVER 5.1.9
