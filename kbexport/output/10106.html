RTF Report Footer Missing

10106

Introduction

When an RTF report is printed, it should display a standard footer including the patient name and date of printing.

Symptom

Nothing appears in the footer of a printed RTF report.

Cause

EncounterPRO Client version 4.05.33 introduced configuration parameters for turning on and off the header and footer of RTF reports.  The parameters are supposed to default to "True", but in some cases they default to "false".

Workaround

Resolution

Run Hotfix 35 (Hotfix 35 RTF Print Footer.sql).
