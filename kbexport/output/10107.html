Message incorrectly attached to patient

10107

Introduction

When a user creates a message from inside a patient chart, the patient context is automatically associated with the message.  When a user creates a message from the "Messages" tab, there should be no patient context.

Symptom

A user creates a message from the "Messages" tab (without any patient context) and sends it to another user.  When the recipient reads the message, it appears to be associated with a specific patient.

Cause

EncounterPRO automatically associates a new message with the current patient, if there is one.  Normally, when the user finishes performing services and returns to the office/room view the patient context in memory is cleared out so that any further operations (messages, utilites, config, etc.) are done without any patient context.  However, when the user clicks on a service ordered for someone else, and then clicks "No" on the "Do you wish to take ownership..." popup, EncounterPRO fails to clear the patient context out of memory so a message created immediately afterwards will be associated with the clicked patient.

If instead of creating a new message the user clicks a service for a different patient, EncounterPRO will recognize the new patient context and correctly clear out the previous patient context.

Workaround

If the user clicks "No" on the "Do you wish to take ownership..." popup, they can immediately log out and log back in again to clear the patient context.

 

Resolution

This problem was fixed in EncounterPRO 4.05.33.7.
