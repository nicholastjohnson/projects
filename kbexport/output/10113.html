Linking databases to Access tables office 2003

10113

Introduction

These step-by-step instructions were made using MS Office Access 2003 with SP1.  Your mileage may vary.

Symptom

Cause

Workaround

Resolution

Creat a blank Access db (mdb) - name it cpr_(version level) and click Create

Click File, Get External Data, and Link Tables

At the bottom of the dialogue box, change the Files of Type: to ODBC Databases ()

Click on New...

Scroll all the way down to the bottom of the driver select box and choose SQL Server, then Next

Name the file data source  cpr_(version), Click Next, and Finished

Leave Description blank, and choose the correct server from the Server: dropdown box.  For internal purposes, choose Techserv.  Click Next

Keep the defaults on the authenticity verification screen and click Next

Check the box for Change the default database to the name of your db (eg cpr_(version) - for internal purposes, usually cpr_405_testing.  Keep other defaults and click Next

Keep defaults on the next page and click Finished, then OK

 

Highlight again the Data Source that you just created in the box and click OK

Check the Save Password box, then click Select all, and then click OK

The Select Unique Identifier box will pop up frequently - keep the tables linking by holding down the Enter key on your keyboard until the last table opens on your screen (usually x_Translation_Set)

Close the table and you now have an Access db lined to the EncounterPro db of your choice.
