How to find a table with missing indexes

10116

Introduction

Sometimes a maintenance job fails and leaves a table without anyindexes.  When this happens, EncounterPRO performance can suffertremendously.

Symptom

A weekly maintenance job has failed and now the customer is complaining that EncounterPRO is frozen or performing very slowly.

Cause


Workaround


Resolution

To find out if any tables are missing their indexes, run the following SQL script:

SELECT t.tablename
FROM  c_Database_Table t
WHERE NOT EXISTS (
 SELECT 1
 FROM sysobjects so
 INNER JOIN sysindexes si
 ON so.id = si.id
 WHERE si.indid < 255
 AND si.indid > 0
 and si.name NOT LIKE '\_WA\_Sys%' ESCAPE '\'
 AND t.tablename = so.name )
 AND t.tablename <> 'c_1_record'
ORDER BY t.tablename

If this query returns any records other than c_1_Record (whichnormally doesn't have an index), then you can rebuild the indexes foreach table by issuing the SQL comand:

EXECUTE sp_rebuild_constraints ''
