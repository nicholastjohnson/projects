PrimeTime HPI configuration

10117

Introduction


Symptom

Using Prime Time HPI (or Instant Medical History (IMH)) at a practice requires some additional configuration.

Cause


Workaround


Resolution

The treatment workplan attached to thePrimeTime HPI treatment mode is called Primetime HPI. (It can beaccessed by going to Configuration, Workplans, set Workplan Type toTreatment, search for Description contains primetime, set specialty toAll, select the workplan, click Display/Edit.) This workplan needs tohave three steps -

1) Service External Source (PrimeTime HPI)ordered for Workplan Owner, flags set to OSPxxx with Service ExternalSource Type of History with all other fields blank.

2) Review - simply a treatment dashboard service (Review HPI) ordered for Workplan Owner, OSPxxx

3) Treatment, Other Observation - compositetreatment, Primetime E and M Credit (ordered with Yes/No treatmentmode), ordered for Workplan Owner, OSPxxx

Primetime E&M Credit Observation structure v2.0 Child treatments:

HPI done in Primetime (one result, normal, - with E&M Credit forCC, HPI-Duration, Modifying facotrs, Signs and symptoms, and Timing)

PFSH done in Primetime (one result, normal - with E&M Credit for all three PFSH bullets.

These are standard, but at the practice's request, add ROS and/orbreak the PFSH down into Family, Social, and Medical histories.

This allows the user to determine how much E&M credit dependingon what the patient entered, which they just reviewed in the previousworkplan step.  If for some reason the HPI is ordered in error,not having the E&M credit automatically applied allows the user toclick "finished" without selecting the result (-) for any of the childobservations and thus getting credit for that history.

Varsion 1.0 - Objective,OBO|RRR|CBN, everything standard. The child is "CC/ PM/ SOCIAL/ FAMILY"and is Subjective, has a perform result of "Done in Primetime IMHx -see attachment" and has the following History coding elements: CC -Chief Complaint HPI - Duration, Modifying Facotrs, Signs and Symptoms,Timing PFSH - Family History, Past History, Social History

If the Quick View or Encounter SOAP Notes don't display the PrimeTime HPI, edit the Standard Quick View HPI Display script to include a Treatment Attachment command
