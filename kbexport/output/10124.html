Normal Abnormal Buttons changing display

10124

Introduction

 

Symptom

Normal/Abnormal buttons not displaying on the History Display screen or they display the same Display Script.

Cause

Lack of definition of Abnormal display script in o_service_attribute.

Workaround

Resolution

Click on the Help button at the top of the screen and click "Display Current Context" to note which service the screen uses (typically "RESULTS_BY_TREATMENT").

Go to o_service_attribute and filter for that service.

If there is an attribute for display_script_id_abnormal, make sure it is different from the not-abnormal one.

If there isn't, copy the display script id for the display_script_id value, filter for it in c_display_script, copy the description, go into the application, Configuration, Display Scripts, Treatment, Display Script Name, past in the name, click on the script on the left, and Copy.  Call it the same name with "Abnormal" at the end.  Then go to c_display_script and sort in reverse order by display_script_id; the top row will be your newly-created script.  Copy its description into the display_script field so they match, then copy the display_script_id.  Go back to o_service_attribute and make a new record for the service with an attribute of display_script_id_abnormal and a paste in the value from c_display_id.

Notes

The “All” button refreshes the RTF area using the display script specified by the attribute “display_script_id”.  The “Abnormal” button refreshes the RTF area using the display script specified by the attribute “display_script_id_abnormal”.  If “display_script_id_abnormal” is not defined, then the buttons don’t show.
