Sendout Test report is not printing out all the ordered send out tests

10125

Introduction

Sendout Test report is not printing out all the ordered send out tests

Symptom

The Sendout test report does not have all of the send out tests ordered for the patient in the encounter..  

Cause

The Sendout test report is dependent on a treatment mode workplan assigned for send out test.

This is a 'legacy' problem. The new enhanced Encounterpro, will have a new treatment type of 'Lab Referral' where this problem will go away. Originally it was thought that labs would be 'in-house', so the default treatment mode is set to Lab Collect Perform. Setting the default treatment mode to 'ong send out or refer out' workplan will not correct the problem, as the report stored proc looks for key phrases in the p_treatment_item. The correct fix is to select the 'alternate' treatment mode of 'ong send out or refer out' 

 

Workaround

For the Ordered /Lab select the alternate treatment mode 'ong send out or refer out' 

Resolution

For the Ordered /Lab select the alternate treatment mode 'ong send out or refer out'
