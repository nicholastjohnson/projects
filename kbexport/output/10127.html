Creating a new customer database

Cause


Workaround


Resolution

1. Restore Database

 a.       If this is being done on the server of a customer, create a database called cpr in SQL Enterprise Manger. Else create a database named after the practice, like cpr_prac_name. Replace "cpr" in the instructions below with the database name.
 

 i.      On the Data Files tab you will have a file name of cpr_Data and you will need to create another file name called cpr_Attachments.  You can put both of these on the same drive or separate them out to different drives.  Because the Attachments can grow rather large at some practices it is a good practice to have a drive just for this file group.
  ii.      On the Transaction Log tab you will have a file name of cpr_Log – again you can keep this on the same drive as the data files or put them on a different drive which is recommended.

 b.      Copy database backup file from CD to Server, if not already done.
 

 i.      Backup file is locate on the CD under New DB Install\DB Backup\405_cpr_010804_Master.exe
NOTE: The file to restore from will change peridoically. Verify which file should be restored from.
  ii.      Once copied to the hard drive unzip

 c.       Bring up the Restore properties window and select from device and go to the path where you restored the above file and select it.

 d.      Now go to the options tab on the Restore properties window.
 

i.      Select force restore over existing database
ii.      Change physical file name to the following (Make changes where necessary for site that have the files groups on different drives)

   1.      (DATA DRIVE):\program files\Microsoft SQL server\mssql\data\cpr.mdf
   2.      (DATA DRIVE):\program files\Microsoft SQL server\mssql\data\cpr_attachments.ndf
   3.      (DATA DRIVE):\program files\Microsoft SQL server\mssql\data\cpr_log.ldf
  

iii.      Now click OK and the database should start restoring.

2. Go to SQL Query Analyzer and run the following scripts

 a.       Location – On CD under New DB Install directory


  i.      “4.05 Demo DB to a live DB.sql”
  ii.      “Practice Configuration Script.sql”
   

1.      This script is not necessary for EncounterPRO to work and can be done later
2.      Before running this script follow the instructions at the top of the script

3. Run EPRO Server and increase the mod level to what is required for running the production build of EncounterPRO.  


4. Add users. For creating users, see article 10119.

5. Add room. For creating rooms, see article 10088.

6. If this is being done on the practice's server, backup the database to the backup folder in program files\Microsoft SQL server\mssql\. Else, backup the database to \\Techserv\New Customer Databases to a file named after the date, like cpr_YYYYMMDD.bak (Include the name of the practice .)
