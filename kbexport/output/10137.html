Medicaid Billing Rules

10137

Introduction

This article explains the default rules for medicaid billing

Symptom

a) How to allow all the sick diagnosis to be billed if WELL/SICK diagnosis are combined:

 By default EncounterPRO allows only first diagnosis to bebilled and suppress the others. To allow everything as billable set thefollowing attribute.

c_Component_Attribute
component_idattribute_sequenceattributevalue
JMJ_MEDICAID151suppress_after_first_sickNO

b) How to bill a sick visit code if a WELL/SICK diagnosis are combined:

If Well/Sick diagnosis are combined by defaultEncounterPRO bills only Well Code. By adding the following attribute itbills a secondary (SICK ) visit code(Est Patient Exam. If they need tochange this from 99212 to something else(like 99202) then theyhave to remove this and manually add the secondary visit code.

c_Component_Attribute
component_idattribute_sequenceattributevalue
JMJ_MEDICAID151well_sick_visit_procedure99212

For standard insurance,

component_idattribute_sequenceattributevalue
JMJ_STD151well_sick_visit_procedure99212

c) Medicaid patients are not generating correct visit codes (i.e., instead WN codes, it goes over as 99 codes)

Version 2

In most cases thecoding_component_id is not set as "JMJ_MEDICAID" in c_insurance tablefor insurance_type='Medicaid'. run the following query in case thereare many records that needs to be updated.

Query:

update c_insurance set coding_component_id='JMJ_MEDICAID' where insurance_type='Medicaid' and coding_component_id is null

Version 4

Inmost cases the coding_component_id is not set as "JMJ_MEDICAID" inc_authority table for authority_category='Medicaid'. run the followingquery in case there are many records that needs to be updated.

Query

updatec_authority set coding_component_id='JMJ_MEDICAID' whereauthority_type='Payor' and authority_category='Medicaid' andcoding _component_id is null
