Patients checking in for wrong encounter types

10139

Introduction

This article explains how to troubleshoot the PM appointment type -> EPRO encounter type mapping issues.

Symptom

a) Well visits checking in as Sick Visits

 

c) Patients always checking in for Sick visits

Cause

  • B_APPOINTMENT_TYPE is the appointment mapping table for mapping the External (PM) appointments -> Epro encounter types. Most of the times either the external appointment code / appointment text is missing from the table or not mapped to a right encounter type.

Workaround


Resolution

a) Find out whether the appointment code / text is missing from our mapping table:

  • Ask the user to check in a test patient for that specific appointment type and go and look in o_message_log "comments" field to see whether it reports the appointment code/ text is not mapped" for that patient#.

b) If  HL7 interface then look in x_encounterpro_arrived table.appointment_reason to see what is the appointment code / text came thru for that patient#.
