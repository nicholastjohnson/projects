Below are the general rules of hotfixes

  • All Hotfixes should be run at all sites where the hotfix shows up on the tab and has not been run before.  After a mod level upgrade, you should go the Hotfix tab and run all hotfixes that have not been run before.
  • We have made a good effort to specify the correct range of mod levels that a Hotfix is valid for.  Too early a mod level and the hotfix might reference a field or object that doesn’t exist.  Too late a mod level and the Hotfix might actually install an obsolete version of a trigger or stored procedure.  If the customer’s mod level is too early for the hotfix then they need to be upgraded.  If the customer has already been upgraded to a mod level past the range for the hotfix, then they already have the hotfix and the problem needs to be investigated.
  • The following hot fixes cannot be run while the practice is using EncounterPRO

  • Hotfix 0001, Hotfix 0011, Hotfix 0016, Hotfix 29, and Hotfix 39
