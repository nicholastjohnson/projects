Message tab shows number but Inbox message list is empty

10154

Introduction

Symptom

The user sees a positive number in red on the "Messages" tab, but when the user clicks on the messages tab, the Inbox message list is empty and an error message appears in the event viewer like this:

SQL ERROR = (137) Select error: SQLSTATE = 42000

Microsoft OLE DB Provider for SQL Server

Must declare the variable '@pl_patient_workplan_item_id'.

Dataobject = dw_message_inbox_list

SQL Statement = 1 execute dbo.sp_get_encounterpro_message   @ps_user_id = 'SUPPORT', @pl_patient_workplan_item_id int = NULL, @ps_ordered_service = 'MESSAGE'

 

Cause

This error to a typo in the datawindow which displays the inbox message list.  On some systems, the datawindow rebuilds the stored procedure call on the fly so the typo does not always cause as an error. The typo was fixed in version 33.6. 

Workaround

Upgrade customer to 33.7 or higher
