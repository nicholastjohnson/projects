v2 Posting TIF to chart gets wrong image

10156

Introduction

When the user posts a tif image to a patient chart, and then posts another tif image to a patient chart, the actual image that gets posted the second time is sometimes the first image, not the second image.

Symptom

When the user clicks "Post" and selects a patient, the thumbnail in the upper left of the "Post To Folder" screen shows the wrong image.

Cause

This problem is caused by multiple users posting documents from thin clients at the same time.

Workaround

The workaround is to set the temporary working file to a different name for each thin client user.  This can be done by running the following SQL script in Query Analyzer:

-----------------------------------------------


INSERT INTO o_Preferences (
 preference_type,
 user_id,
 preference_id,
 preference_value)
SELECT 'SYSTEM',
 '!!' + c.computername,
 'temp_image',
 'tmpimg_' + SUBSTRING(c.computername, CHARINDEX('\', c.computername) + 1, 1000) + '.tif'
FROM o_Computers c
WHERE CHARINDEX('\', c.computername) > 0
AND '!!' + c.computername NOT IN (
 SELECT user_id
 FROM o_Preferences
 WHERE preference_type = 'SYSTEM'
 AND preference_id = 'temp_image')
