How to update siblings phone number Medical Manager from it's guardian record

10158

Introduction

This article explains how to automatically (thru interface) update the siblings phone number from it's guarantor records.
Supported from EPROServer version 4.05.2 and up.

Symptom

Siblings phone number & address are not updated

Cause

 Disabled or missing attribute in c_Component_Attribute

Workaround


Resolution

a) To make the siblings' demographics from the"gurantor" (their guardian) update automatically in the future, set the following attribute inc_component_attribute table.

c_Component_Attribute
component_idattribute_sequenceattributevalue
MSG_HND_MEDMAN_CHECKIN1000084update_siblings_phone_from_guarantorTrue

b) To allow the users to update these siblings phone number& address manually thru EncounterPRO, set the aboveattribute value to 'FALSE'.

 

To update the phone#,address to all the pre-existing siblings from a Gurantor run the following query:

DECLARE @ls_billing_id varchar(20),@ls_phone_number varchar(32),@ls_address_line_1 varchar(40),@ls_address_line_2 varchar(40), @ls_cityvarchar(40), @ls_zip varchar(10), @ls_state varchar(2)

DECLARE lc_patient_guarantor CURSOR FOR
SELECT billing_id,phone_number,address_line_1,address_line_2,city,state,zip
FROM p_Patient where billing_id like '%.0%'

OPEN lc_patient_guarantor
FETCH lc_patient_guarantor INTO
 @ls_billing_id,
 @ls_phone_number,
 @ls_address_line_1,
 @ls_address_line_2,
 @ls_city,
 @ls_state,
 @ls_zip
WHILE (@@FETCH_STATUS = 0)
BEGIN

 Update p_patient
 set phone_number = @ls_phone_number,
 address_line_1 = @ls_address_line_1,
 address_line_2 = @ls_address_line_2,
 city = @ls_city,
 state = @ls_state,
 zip = @ls_zip
 where billing_id like (substring(@ls_billing_id,1,PATINDEX('%.%', @ls_billing_id)))+'%'


 FETCH lc_patient_guarantor INTO
 @ls_billing_id,
 @ls_phone_number,
 @ls_address_line_1,
 @ls_address_line_2,
 @ls_city,
 @ls_state,
 @ls_zip

END
CLOSE lc_patient_guarantor
DEALLOCATE lc_patient_guarantor
