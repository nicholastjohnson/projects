GRITS Configuration addendum

10163

Introduction

Changing the default treatment workplan for the treatment type Immunization and changing the Close workplan for encounter types in which immunizations are normally given (or all encounter types, at the practice's discretion.)

Symptom


In version 4, the autiomatic GRITS report does not work after the standard GRITS configuration steps have been performed.

Cause


Workaround


Resolution

Check to make sure that everything you need is in the DB:

  1. Go to Configuration, Observations, Other Observation, and search for "GRITS Vaccine Eligibility".  If you can not find it, see KB article 10352. (Make sure that you also search under Treatment Type: All. If you find it under All but not under Other Observation, then from within the observation's config, go to Treatment Types and add the treatment type Other Observations and verify that this is the onlyTreatment Types listed here.)
  2. Go to Configuration, Workplans, change workplan type to Treatment, search for description "Vaccine for GRITS Report" (be sure to click on the All/specialty toggle to do a thorough search)  If you can not find it, see KB article 10352.

Change which workplans are used for Vaccines and closing the encounters:

  1. Go to Configuration, Treatment Types, Immunization, Edit, and change the default treatment workplan to either "Vaccine for GRITS Report" or "Vaccine for GRITS Report with Admin Codes" depending on whether they're used to having the treatment workplan be simply "Vaccine" or "Vaccine with Admin Codes"
  2. Go to Configuration, Encounter Types, and click on the Well Child Established Visit (EST Patient Well Visit) on their short list and Edit.  Click on the box beside Close Workplan and choose "Bill Encounter with Grits File Selection."  Do the same for the Well Child New visit (New Patient Well Visit).
