Users are showing in wrong Exam Room on Office View

10169

Introduction

 

Symptom

Users log into Exam Room 1 but show on the office view that they are logged into Exam Room 2.

Cause

Workaround

Resolution

The logic of EncounterPRO when setting up a Workstation or Thin Client session to open to a certain Exam room is the following:

The Room ID in o_rooms is set up to use a certain Windows Logon defined in (room_computer) which is drived by what is listed in o_users. For example if the user is physically in Exam Room 2, but with the Windows logon of Exam01 it will show the user in Exam Room 1 on the Office View Screen.

This usually happens when a workstation is connecting to EncounterPRO with the incorrect user name and password. Advise practice to connect to EncounterPRO with the correct user Name to Windows to prevent this problem.
