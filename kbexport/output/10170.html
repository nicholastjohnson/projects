Configuring the EPAutoCheckin Self-Checkin program

10170

Introduction

The EPAutoCheckin program allows patients to use a terminal in the waiting room to check themselves in for services.

There are many configuration options available for EPAutoCheckin once it is installed.  The first time you run EPAutoCheckin you get a screen offering configuration options, but if you need to change them after that it must be done in c_Domain in the EncounterPRO® database.

Resolution

To install this program, first make a folder here: "C:\Program Files\JMJ\Encounter PRO\EPAutoCheckin\".

Download the program to this folder from here: "ftp://ftp.jmjtech.com/EP_Upgrades/Production/EncounterPRO/SelfCheckin/EPAutoCheckin.exe" then make a shortcut from this executable to JMJ Tools on the desktop.

 

This is the configuration screen you see when you first start EPAutoCheckin:


Server is the name of the server on which your EncounterPRO® database resides.
Database is the name of your EncounterPRO® database.
Office ID is the office in which you want the patient encounters to appear when they check in.
Created By controls what goes into the created_by field in p_Patient_Encounter.  If you are not sure what to do for this, do not change the default value.
Max Results controls the maximum number of matching patients that will be displayed on a search.  If a search returns more hits than the maximum specified here, the user will be asked to fill in more information to narrow the search results.  The default is 1 so that a patient will not be shown a list of information about any patients other than himself.
Idle Timeout is the number of seconds without user interaction that EPAutoCheckin will wait before returning to the initial search screen.
Require Last Four Digits of SSN, if checked will prevent a search from returning any results unless the last four digits of a social security number have been entered on the search screen and those digits match the SSN in the patient record.

Next, go to c_domain. For each encounter type you want patients to be able to check themselves in under, enter the domain_id of AUTOCHECKIN_ETYPE, a domain_sequence, and set the domain_item to the encounter_id (from c_encounter_type) and set the domain_item_description to the encounter type description.

After the initial configuration, if the configuration needs to be changed you may do so directly by looking in

Notes

The server and database options are stored in config.xml in the folder in which EPAutoCheckin.exe resides.  If the config.xml file is destroyed, the configuration screen will appear the next time EPAutoCheckin is run.
