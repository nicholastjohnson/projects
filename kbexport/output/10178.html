Configuring the EncounterPRO Agent

10178

Introduction

The multi-threaded EncounterPRO Agent has many configurable values to tweak performance and timeout options.

Symptom


Cause


Workaround

Resolution

Configurable Values:

All configurable values are stored in c_Domain with domain_id = 'EPROSERVERSERVICE'. The domain_item is the name of the configurable value and domain_item_description is its value. None of these values need to be in c_Domain. EncounterPRO Server will assume the specified default for any values not found in c_Domain.

RemotingPort

The port used for hosting .NET Remoting support (default 8089) new in EproServer 5.1.4

MasterMemMaxMB

The maximum amount of memory in MB that EP Agent will be allowed to use before an internal restart (default 150) new in EproServer 5.1.4

TaskQuerySleepTime

The time to sleep between queries to the database for waiting Tasks. (milliseconds) (default 5000)

TaskThreadMax

The maximum number of Task threads per o_Server_Component entry. (default 5) (default 3 in EproServer 5.1.9 and later)

TaskMemMaxMB

The maximum amount of memory in MB that any Task may consume before being killed. (default 150) new in EproServer 5.1.4

TaskQueryWaitTime

The time to wait before re-querying when there are already queued tasks. (milliseconds) (default 60000)

TaskTimeout

How long to wait for a Task to exit before assuming it has hung and killing it. (milliseconds) (default 600000)

ShortTaskTimeout

How long to wait for a Task to exit before attempting to move it to an extended pool.  If the extended pool is full, the task will be terminated. (milliseconds) (default 60000) new in EproServer 5.1.9

TaskProcessPriority

The priority of Task processes. (default 2) (default 3 in EproServer 2.10 and later)
1 = Idle
2 = BelowNormal
3 = Normal
4 = AboveNormal
5 = High

ServiceExitInterval

see ServiceExitMax (milliseconds) (default 3600000)

ServiceExitMax

The maximum number of times a static service is allowed to exit within the period specified by ServiceExitInterval before it is no longer restarted when it exits. (default 10)

ServiceMemMaxMB

The maximum amount of memory in MB that a static service will be allowed to use before being killed (default 150) new in EproServer 5.1.4

ServiceProcessPriority

The priority of static service processes. (default 2) (default 3 in EproServer 2.10 and later)
(see TaskProcessPriority)

StopProcessTimeout

Time to wait for processes to exit when the service is stopped before forcing them to exit. (milliseconds) (default 30000)

StartRetryMax

Maximum number of startup retries after a database connection has been established. (default 5)

VerboseLogging

Enable verbose logging (default 1) new in EproServer 2.10
0 = Off
1 = On
