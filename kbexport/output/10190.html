Cannot insert duplicate key row in object 'c_Observation' when updating Config Level

10190

Introduction

 

Symptom

This happens when the observations ID are incorrect for the GRITS observations. Below are the errors msgs that will usually occur

DECLARE @local_error int, @temp_string VARCHAR(40)

SET @local_error = 0

BEGIN TRANSACTION

SQL ERROR = (2601) SQLSTATE = 23000

Microsoft OLE DB Provider for SQL Server

Cannot insert duplicate key row in object 'c_Observation' with unique index 'idx_c_Observation_id'.

4.05.2.2 (2/10/2005) >>> u_sqlca - (execute_script()) Error Executing SQL Statement:

4.05.2.2 (2/10/2005) >>> u_sqlca - (upgrade_content()) Error executing upgrade script #866

 

Cause

The observation ID's are incorrect due to a previous copy and paste done from the master to client's database.

Workaround

Resolution

To solve this problem you will need to modify the obeservation ID's in c_observation. They should show in the below order.
