GRITS Installation and Configuration

10191

Introduction

This article explains the normal way to install GRITS in EncounterPRO version 4.

Symptom

Cause

Workaround

Resolution

1- First confirm the practice knows about GRITS. Whether they do or not, send them an email describing the feature along with contact info for a GRITS representative. In this email, include the practice's info and copy the GRITS contact on the email.

Example

2- Run this query against the practice's database

3-  From our ftp server, go to EP_Reports\4.0 and download and install "Report Setup.exe" to the server in cprfiles\GRITS. You will need to supply the server name and database name to the installer. When installing Setup, choose only these components: 

GRITS Batch
GRITS File Creation
 
4- In epro, go to Config, Reports, General, Grits, Default* :
 Vaccine Eligibility: in list of observations, find and select GRITS Vaccine Eligibility ;
 File output directory:\\(servername)\cprfiles\grits\;
 server: (servername);
 db: (db name)
 sending organization: This is a numeric code the practice needs to get from Lisa Roberts, which the practice must then give to us;
Next,
 Config, Reports, Encounter, Grits, Default :
 (same as in the General tab)
 
5- Run this query against the practice's database.
NOTE, this query should do the same thing as what is described in KB Article 10163
 
6- Test the reports. If they works, inform the practice and train the practice on how to use them.

*If at this point nothing happens and you get this error in the Event Viewer:
u_reports_edit - (u_report_edit) Error getting report component (GRITS_REPORT)
Then see KB 10506

Notes

1- See also \\Dellserv\Common\Support\Knowledge Base\In House Training Documents\Integration\GRITS

for all this information above

 

2- Please update the file GRITS status.xls in the afore mentioned directory whenever a practice's status with the GRITS report has changed.
