Installing and Configuring Immunization reports

10192

Introduction

This article is about installing and configuring the Immunization reports for version 4.

Symptom


Cause


Workaround


Resolution

  1. Download this file"ftp://ftp.jmjtech.com/EP_Reports/4.0/Extract ReportInstaller.exe" tothe \install\jmj folder on the practice's server.
  2. Run the file. Specify the practice's server and epro database name. UseIntegrated Security.Double click the checkbox next to the immunizationreport name and click Finished.
  3. Ifthe report was installed successfully, go to EncounterPRO, Config,Reports, Patient, (the immunization report name ), Default.
  4. Verify thereport filename and macro name are filled in.  For each vaccine,click the blank grey square and select all vaccines that match.
  5. Go to the folder in CPRFiles where the report is installed and make the file read-only.
  6. Test this report on a patient and if it succeeds, inform thepractice that it is ready for them to use. Remind/inform them that if they want to save changes of this copy of thereport for this patient, they'll have to save the report under anothername, preferably something indicating the patient's naem or billing idand the date, and somewhere other than the location of the originalreport file.

Notes

Fl Immune report

  • This report prints only the field data, none of the field names or any other text as it is meant for pre-printed forms.
  • When specifying the vaccine for MMR separate, use any vaccinethat treats Measles OR Mumps OR Rubella, but not all 3. When specifyingthe vaccine for MMR complete, use vaccines that treat all 3.
NH MA Immune report
  • This report includes fields for various observations that must be specified in the report configuration.
  • The observation Other must be filled in by the practice at runtime (unless the Other field will always be the same in which case thiscan be configured after the report is installed at the practice).
MD School Health Assessment
  • This report includes vaccination and observation data (like height, weight, blood pressure, etc).
