Spirometer interface EncounterPRO configuration

10196

Introduction

Below is an explanation of setting up EncounterPRO to accept files generated by a spirometer.  As of 2/26/05, the only spirometer that will work within EncounterPRO is the Brentwood/Midmark IQMark digital spirometer.  There will be other spirometer devices that we will eventually interface with, but they make/model of spirometer should not effect the EncounterPRO observation configuration described below.

 

NOTE: Article #10146 - Setting up Spirometer integration is an article outlining the installation of the spirometer device, and is part of the total installation.

Resolution

  1. Identify all of the observations that the practice will use to obtain spirometry results.  There is typically one observation that the practice will use consistently, but make sure not to miss any other cpt codes that they will use.
  2. Order the observation(s): 
     
  3. Configure the observation.  After ordering the test and when you are in the Drilldown menu, select other options, configure.  Note: Users can also configure observations from the configuration tab. In this case, go to Configuration, Observations, Test, Spirometry*

  4. Configure attachment results by clicking on Perform, Attachment, click on the big blank grey square, New Result.

  5. Add a new result to the attachment result (ex. Spirometer, Spirometer results, Spiro results, etc...), i.e., under the text box for "Result from External Source", type in Spirometer Results. Set the external source to "Spirometer, <Any>." 

  6. Finally, setup the fat client's external sources to acknowledge the device.  Go to configuration; external sources, and select spirometer.  After doing these configuration steps, making sure that all of the correct spirometer software is installed, and ensuring that the spirometer is functional, you should see a green line in the result that was created in the observation.

  7. If there is a problem with any step in the process you will notice a red line (shown below), indicating that there is something wrong in the setup.
