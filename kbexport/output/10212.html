Cannot create new results for observation in version 2

10212

Introduction

An observation can only have 32767 results created for it.  Once this number is reached for any given observation, no more results will be able to be created.

Symptom

If you click on an observation and click on Result and try to enter an result on the fly it throws this error:

u_sqlca - (check()) SQL ERROR = (515) Cannot insert the value NULL into column 'result_sequence', table 'cpr.dbo.c_Observation_Result'; column does not allow nulls. INSERT fails.

It also does the same thing in configuration.

Cause

The result_sequence field in c_Observation_Result is a 2-byte integer.  2-byte integers can have a maximum value of 32767.

Workaround

We cannot fix the existing maxed-out observation, but we can create a new observation which will look and act just like the old one but without all the old results.  Then we can replace the old observation with the new one on all the shortlists and treatment lists.  The logic to do all this is in the sp_replace_observation stored procedure.

First, apply Quickfix 2.21.44.04 to create the "sp_replace_observation" stored procedure.

Second, determine the observation_id of the maxed-out observation and executre the sp_replace_observation stored preocedure like this:

EXECUTE sp_replace_observation '<<observation_id>>'

Where <<observation_id>> is the maxed-out observation_id.

Resolution

This issue is fixed in EncounterPRO version 4
