Backup Check Warning

10215

Introduction

Each time a user logs in, EncounterPRO will check to see if a recent database backup has been performed, and a warning will be displayed to the user if no recent database backup is found.

Symptom

Sometimes users get this message even when a recent backup has been performed.

Cause

EncounterPRO uses the following query to determine the last time the database was backed up:

SELECT max(backup_finish_date)
FROM msdb..backupset
WHERE database_name = '{database}'

where {database} is replaced with the appropriate database name.

This only works when the backup process logs records into the msdb database.  The SQL Server "BACKUP DATABASE" command will always log into the msdb database.  The msdb database is not required for normal SQL Server operations, so if it has become corrupt or missing then users might not know other than the backup check warning.  Also, there are third party backup/recovery systems which might not log records into the msdb database.

Workaround

The backup warning check can be disabled by adding a record to o_Preferences where the preference type is ‘SYSTEM’ and the preference_id is ‘disable_backup_check’ and the preference value is ‘True’.

 

Resolution

Make sure the msdb database is operational and that the backup process is logging records into it.
