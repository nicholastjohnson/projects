RTF Reports show Page of X in footer

10218

Introduction

 

Symptom

When the user views an RTF report, and then clicks "Edit" to bring up the report in an editor such as Microsoft Word, the page number in the footer of every page reads "Page: of X" instead of having the appropriate numbers.

Cause

When an RTF report is generated, the page numbers are not know because they can change based on the size of the display control and other dynamic factors.  The numbers are actually calculated immediately before printing to insure their accuracy.  The "Edit" button on the report display screen writes the RTF report to a file without calculating the page numbers.

Workaround

The "View" button on the bottom of the RTF report display screen changes the display to show the entire page, including header and footer information.  The "View" button does calculate the page numbers so that they appear correctly on the screen.  If the user therefore clicks "View" and then clicks "Edit", the page numbers will appear in the document correctly.

Resolution

This problem has been fixed in EncounterPRO version 4.05.34.
