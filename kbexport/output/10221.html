No growth vital data is displayed under show data on development tab

10221

Introduction

 

Symptom

Growth data appears to be missing from the "Show Data" screen on the Devel tab in the patient chart.  An error message like this one is report to the event viewer:

Unknown Caller - (SYSTEM ERROR) errornum:3
message :Array boundary exceeded at line 155 in function refresh_graphs of object u_develop_combo.
where   :u_develop_combo
object  :u_develop_combo
event   :refresh_graphs
line    :155

Cause

This problem is caused by a logic error in EncounterPRO which causes an error when trying to display growth data entered for the day the patient was born.

Workaround

Epro Version 2:  A workaround which causes EncounterPRO to display growth data entered for the childs birthday as if it were entered on the day after his/her birthday is in QuickFix 2.21.44.06.SQL on the version 2 production CD.

Cd2.21.44\EncounterPRO\scripts\2.21\Updates

 

Resolution

Epro Version 4: This problem is fixed in Epro Client version 4.05.33

Notes

Version 2 & Version 4: sometimes there are no errors shown in Event Viewer and still no data displayed in growth curve or show data. this might happen if result_date_time is null or an incorrect data type has been entered for either of 'WGT','HGT','HC' records for a given patient account. run the below query to see whether it returned any rows.

Version 2:

select * from p_objective_result where observation_id in ('WGT','HGT','HC') and result_date_time is null

Version4:

select * from p_observation_result where observation_id in ('WGT','HGT','HC') and result_date_time is null

if the above query returned some rows then try the following update query to fix the result_date_time with treatment begin date.

update p
set p.result_date_time=t.begin_date
from p_observation_result p,p_treatment_item t
where p.treatment_id = t.treatment_id
and p.result_date_time is null
and p.observation_id in ('WGT','HGT','HC')
