Sorting items on the office and room views

10222

Introduction

Prior to build 34, the items on the office an room views were always sorted by patient name, and then within patient name ther were sorted by the age of the item with the oldest at the top.  The waiting room always sorted by waiting time descending.

Beginning with build 34, the office view, room view, and waiting room tab may be optionally sorted by patient name, ascending, or  the age of the encounter, either ascending or descending.

Symptom

Cause

Workaround

The sort order for each room is controlled by records in the o_Preferences table.  The desgniation of a sort order for a room affects all the items in that room whether they appear on the office view or the room view.  There is currently no user interface for creating/editing room-specific records in the o_Preferences table, so this feature may only be enabled by qualified support personnel.

To designate the sort order for a specific room, add a record to o_Preferences as follows:

preference_type = 'SYSTEM'
preference_level = 'Room'
preference_key = <Room_id>
preference_id = 'sort'
preference_value = 'Patient' or 'Encounter Ascending' or 'Encounter Descending'

The Waiting Rooms tab is not able to sort different waiting rooms differently.  Therefore, to set the sort preference for the Waiting Room tab, use '$Waiting' as the <Room_id> instead of any specific waiting room id.

The default sort preference for waiting rooms is 'Encounter Descending'.  The default sort preference for all other rooms is 'Patient'.
