Configuring and using the Chart Export service

10237

Introduction

Build 34 introduces a new service for exporting an entire chart into a set of files.  The contents of the files is controled by service attributes and display scripts.  In addition, the attachments can be included in the export.

Installing Chart Export

 

The Chart Export database objects will be installed automatically with Mod Level 127.  They can also be added to Mod Level 126 by running Hotfix 129.

 

Configuring Chart Export

 

The Chart Export service has the following configuration parameters:

 

Attribute

Default

Meaning

default_path

c:\temp\PatientCharts

Root directory where files will be placed.  A subdirectory matching the patient’s billing id will be created and all files will be placed in the subdirectory.

patient_reports

None

Comma delimited list of display script ids to run against the patient.  The output of each display script is saved into a separate file.

encounter_report

None

Single display script id to perform for each encounter in the patient chart.  The output for each encounter is saved into a separate file, with the encounter date appended to the file name.

 

The Chart Export service can perform multiple reports for the patient and a single report for each encounter.  Examples of patient reports might be a problem list, an immunization history, a PFSH summary, etc.  If no display script id is specified for the patient_reports attribute, then no patient level reports will be produced.  Likewise, if no display script is specified for the encounter_report attribute, then no encounter reports are produced.

 

As with any service attributes, the attributes can be specified in the o_Service_Attributes table (recommended in this case), or overridden in the menu or workplan item configuration.

 

The Chart Export service is a patient-level service and can be added to any menu or workplan item which has a patient context.  It is recommended that the Chart Export service be added to the menu at the bottom of the Patient Demographics screen.

 

Running Chart Export

 

When the Chart Export service is performed, a screen is displayed showing which reports will be run for the patient, which encounter report will be run for each encounter, and whether to export all of the attachments.  All reports and attachments default to “Yes” meaning to produce them.  The user may change any report or the attachments to “No” meaning to exclude it from the export.  The name of the reports comes from the name of the Display Scripts specified in the service attributes.  The Display Script name is also used to name the file that is created for each report.  Each report file is saved and an RTF file in the subdirectory created from the patient billing id.

 

If the attachments are included in the export, then another subdirectory is created under the billing-id subdirectory called “Attachments”.  Then under the Attachments subdirectory another subdirectory is created for each folder in the patient chart which contains attachments.  Any attachments which are not in any folder in the patient chart will be stored in a subdirectory called “No Folder”.  The attachments are saved in whatever form they were originally imported.
