Extracting New Existing Report Requirements from Customers

10238

Introduction

This article explains what to ask from a customer and what JMJ development requires for any new report requirements.

Symptom

a) The customer is looking for some report that is not found in EPRO report system

b) The customer wants to add more details in existing reports.

Cause

Workaround

Resolution

Requirements for New / Updating existing report

1) If this is a brand new report, then get a report template(with all the details they need) from the customer and create an issue for "Development new". By reviewing the requirements, JMJ development notify the support how much it'll cost ($$) to implement and what report release that would be available.

2) If this is an enhacement or add-on for our existing reports, we still need the report template on what needs to be added. Based on the requirments, we again decide whether it's something we customize only for this customer or make it as a permanent change to all our customers. This might or might not cost and again JMJ development will notify after reviewing the details.
