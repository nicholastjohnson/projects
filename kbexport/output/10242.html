Error Accessing .NET Library Through COM Interop

10242

Introduction

This article does not apply to the following product versions or those released later:

EproLibNET 102.7
EncounterPRO Client 4.05.33.19
EncounterPRO Client 4.05.34.9
EncounterPRO Server 2.7

For these product versions and later, please refer to the following article to register EproLibNET.dll, EPImageViewer.dll, or EPImageControl.dll
Manually register EproLibNET.dll, EPImageViewer.dll, EPImageControl.dll

Symptom

An error like:

Error creating EPImageControl object
or
Error creating EPImageViewer object
or
Error creating EproLibNET object

Cause

The .NET library referenced in the error message either does not exist or has not been properly registered for COM Interop

Workaround

  1. Install or Repair the latest production release of EproLibNET.  If the problem is not solved, proceed to step 2.
  2. Verify that the following files exist by performing a file search on all drives--be sure to allow the search to complete to get all results:
    1. EproLibNET.dll
    2. EPImageControl.dll
    3. EPImageViewer.dll
  3. Delete any copies of the above files found an any location other than C:\Program Files\Common Files\JMJTech Common\EproLibNET
  4. Download RegisterForInterop.vbs and drag the remaining search results onto the .vbs file.  You should recieve four (4) confirmation messages per file.
  5. Test to make sure the problem is solved.  If you have problems viewing images after completing these steps, refer to Image Viewer does not display.

If you encounter an error when running the VB Script, you may need to install Windows Scripting.
