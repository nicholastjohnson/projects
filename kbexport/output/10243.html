Patient portrait not showing in EPRO with a database storage scheme

10243

Introduction

This article explains how to troubleshoot and resolve issues of patientattachments not showing in EncounterPRO where attachments are stored inthe database instead of CPRFILES

Symptom

  1. One or a few patient attachments are not displaying. This error is found in Event Viewer when attempting to view the attachment:
    •  u_component_attachment_generic - (open_attachment()) Error getting attachment file
  2. After finding the filename in p_attachment, the filename of the attachment is not found / stored in CPRFILES
  3. No pt attachment files are stored in CPRFILES

Cause


Workaround

  • Download and extract this program:
ftp.jmjtech.com\JMJSupport\Tools\EPAttchExtractor.zip
  • Specify the server and database and an output folder. Click on Edit query to specify the attachment id (or other criteria).
  • Then with the attachment file, re-import the attachment into thept's chart. Make sure to replace the new attachment's description withthe old attachment's description.

Resolution


Notes

In EPAttchExtractor.exe, use the option "Update" ONLY ifyou are sure you know what you are doing. This will replace the imagefile in the database with the working image file and not make anyupdate to any attachment progress log table.
