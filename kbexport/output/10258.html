Error AHC object not connected

10258

Introduction

This article explains why this error is reported

Symptom

  • TransportIn throws this error

Error calling external object function connect at line 22 in function ahc_connect of object u_component_incoming_filecopy_hl7.
where   :u_component_incoming_filecopy_hl7
object  :u_component_incoming_filecopy_hl7
event   :ahc_connect
line    :22

  • or TransportOut throws this error

u_component_serverservice_outgoing - (timer) Error in timer_ding function (Error calling external object function connect at line 23 in function ahc_connect of object u_component_outgoing_filecopy_hl7.)

Cause

a) Incorrect "AHC" dll files registeration

b) In  registry "HKEY_LOCAL_MACHINE->SOFTWARE->AHC->APPLICATIONS and the "client" property is set to the incorrect computer name.

c) In  registry "HKEY_LOCAL_MACHINE->SOFTWARE->AHC-> and the "directoryserver" property is set to the incorrect computer name.

d) Permission error with Transport In logon user

Workaround


Resolution

a) Re-register the "AHC" dll files in "System32" folder

b) Check all the instances in  registry "HKEY_LOCAL_MACHINE->SOFTWARE->AHC->APPLICATIONS and make sure the "client" property is set to the correct computer name that is running the encounterpro services.

c) In  registry "HKEY_LOCAL_MACHINE->SOFTWARE->AHC-> and the "directoryserver" property is set to the correct computer name that is running the encounterpro services.

d) Make sure the Transport In/Out logon user is privileged with "local Administrator"
