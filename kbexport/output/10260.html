Prescription description gets cut off when prescription is modified

10260

Introduction

 

Symptom

When a prescription is modified, some of the description from the original is cut off and not included in the modified prescription.

Cause

Only the first 80 characters of the treatment description are stored in the p_Treatment_Item table.  Any descriptions longer than 80 characters are stored in the p_Treatment_Progress table.  When a treatment is modified, a copy of the p_Treatment_Item record is created but, prior to mod level 127 (or Hotfix 60), the associated p_Treatment_Progress record was not copied so the treatment description appeared to be truncated to 80 characters.

Workaround

If the customer database is below mod level 119, upgrade the database.

If the customer database is between mod level 119 and 126 (inclusively), apply hotfix 60.

Resolution

This is fixed in Mod Level 127 (dev issue # 5814)
