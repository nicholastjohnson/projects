Shrink Log File-not enough disk space

10262

Introduction

One shrinks the log file because it may be taking up to much disk space.

Symptom

Will get the following or a similar error message in event viewer:

Event Type: Error
Event Source: MSSQLSERVER
Event Category: (2)
Event ID: 17052
Date:  5/4/2005
Time:  8:13:45 AM
User:  N/A
Computer: DUMONT-WIN2K
Description:
Error: 9002, Severity: 17, State: 6
The log file for database 'CPR' is full. Back up the transaction log for the database to free up some log space.
Data:
0000: 2a 23 00 00 11 00 00 00   *#......
0008: 0d 00 00 00 44 00 55 00   ....D.U.
0010: 4d 00 4f 00 4e 00 54 00   M.O.N.T.
0018: 2d 00 57 00 49 00 4e 00   -.W.I.N.
0020: 32 00 4b 00 00 00 04 00   2.K.....
0028: 00 00 43 00 50 00 52 00   ..C.P.R.
0030: 00 00                     ..     

Cause


Workaround


Resolution

To Shrink the log file you run the following script on the database through Query Analyzer - (for possible error messages, look below for workarounds):

Shrink the DB by running "Shrink Transaction log.sql" Shrink Transaction Log

-------------------

If you get a error message regarding cpr being full when running the script you may need to run the following script first then run the original one again:
dump transaction cpr with truncate_only
