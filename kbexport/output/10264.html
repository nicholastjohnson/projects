EncounterPRO freeze while clicking an insurance tab in chart screen

10264

Introduction

this article explains why encounterpro freezes by clicking insurance tab

Symptom

EncounterPRO freeze while clicking insurance tab for certain (or all) patients

Cause

a) this is because somehow the progress_key in p_patient_progress is left NULL for progress_type='insurance notes'. run the following query to see whether there are any records returned.

SELECT * from p_Patient_Progress where progress_type='insurance notes' and progress_key is null and current_flag='Y'

if this query returned any results then go and set the current_flag to 'N' to fix this. if this happened on very recent dates then log this as a bug (if you could reproduce) in dev issues.
