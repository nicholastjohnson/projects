Re-Posting charges from checkout cancels all the requests

10267

Introduction

This article explains why the reposting of charges gets a 'Cancelled' status

Symptom

  • Clicking on "retry posting" cancels the request even if the encounter is valid with billable items
  • Clikcing on "retry posting" completes the request but not generating a bill.

Cause

'Cancelled' Status

There are 3 possible reasons for this:

  1. The encounter is not billable
  2. The user selected "hold"  billing option. this causes the encounter set to "non-billable" until they approve the billing.
  3. User suspended the billing.
  4. billing component is not set for that encounter office in c_office table.
To unsuspend billing. Recheck in patient and then exit encounter and it will prompt you to unsuspend the bill. If you select Yes it will unsuspend the bill. You will then have to re-post the encounter.

'Completed' Status

If the status is completed most of the times a valid bill must be generated and send out. if it didnt then there is ONLY ONE possible reason for that. The encounter owner must have choosen "suspend the bill" [find a encounter progress record send_billing_suspended ="T"] and there must be an item on his/her todo box to complete the chart. In case they completed the todo item [the popup will ask again whether they want to release the bill now or cancel] and didn't allow the billing to go, then it will never sent the bill and always "complete" the service with all the re-post requests.
