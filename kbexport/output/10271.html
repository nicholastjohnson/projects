Creating Services

10271

Introduction

This article explains how to create Windows services like "EncounterPRO Service"

Symptom


Cause


Workaround


Resolution

  1. First you need to create the users that will run the services
    1. The password for the user is !$Epro&&
    2. User must be a member of the domain administration

1.      eprojmj

  1. Next you will create the actual service
    1. Unzip the file Create Epro Services Version 4.zip and pull out the appropriate .bat file.  This file is located on the CD under New DB Install.

                                                              i.           HL7 Integrations – Millbrook, Centricity, Elligence, MARS,M&M, Compusence & Vantagemed

                                                            ii.           Message Handler Integrations – Fox Meadows, Medisys,Raintree & Lytec

                                                           iii.           Contact JMJ Support if your integration is not listedabove and you are still not sure which file to use.

    1. If you have EncounterPRO installed on any other drive than the drive D with the default install path of “Program Files\JMJ\EncounterPRO” then go to i, otherwise execute the appropriate .bat file.

                                                              i.           Right click on the .bat files and select edit.  Change the path for each line to the paththat the EncounterPRO client is installed.

    1. Once the services are created you will have to go to one of the services and re-enter the passwords to set the service rights before any of the services will work.
  1. Now remove the Dependencies.reg file from the Epro Services Version 4.zip file.
    1. Execute this file and select yes and then ok.  This will make it to where the EncounterPRO services will not start until after the SQL Server service starts.
