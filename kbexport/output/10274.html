GRITS Login failed error

10274

Introduction

This article explains what to do when a user gets a login error when running the GRITS report.

Symptom

  1. While attempting to run the GRITS report from EncounterPRO, some users (but not all) get the following error:

Cause

GRITS requires each user of the GRITS reports to have a SQL login.

Workaround


Resolution

Give the Windows users SQL permissions:
Go to Enterprise Manager / SQL Server, local->Security->Logins,New Login, browse for user named (servername)\EproUsers, add user,default db=cpr

Notes

See also support issue 19584
