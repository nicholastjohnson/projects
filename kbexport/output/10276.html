Cannot clear read-only flag in database options

10276

Introduction

 

Symptom

When recovering a database backup file or transaction log backup file with the "Recovery Completion State" set to "Leave Database read-only and able to restore additional transaction logs", the database is indeed left in the read-only state, but attempts to clear the read-only flag in database properties results in an error message stating that the database is in "Warm Standby" mode.

Cause

When recovering a database backup file or transaction log backup file with the "Recovery Completion State" set to "Leave Database read-only and able to restore additional transaction logs", the transactions in the database are left in a state where they are neither committed nor rolled back.  This is necessary in order to recover subsequent transaction logs, but must be cleared before the database can be used.

Workaround

Resolution

To finalize the transactions in a "Warm Standby" database so that the read-only flag can be cleared, run the following command against the MASTER database:

RESTORE DATABASE <database name> WITH RECOVERY
