How to setup EncounterPRO to automatically upload the vaccine data to state immunization registry

10279

Introduction

This article explains how to setup a EncounterPRO workplan that automatically uploads the vaccine data to state registry at the end of an encounter

1) Add a service to the "Close Encounter Workplan"

The “Close Encounter Workplan” is usually called “Bill Encounter” and it’s an Out-Of-Office Encounter workplan.

 

Add a service to step 1 as follows:

 

Select “External Source” as the service

Select “Report” as the External Source Type

Select “Immunization Report” as the External Source

Set the Send To Addressee attribute to the Addressee ID of the appropriate immunization registry (See Jim M or Chuck W).

After the service is created, click on the service and select “Ordered For”.  Select “JMJ Service” from the “System” list as the “Ordered For” user.

Click on the service again and select “Edit Description”.  Set the description to “Send Immunization Data to Registry”

3) Setup a user id and password to make a webservice call to jmj integration engine

Go to configuration->preferences and select computer configuration->globals.

Set the "EpIE User" preference to the Customer ID for that customer.

Set the "EpIE Password" preference to the EpIE password for that customer (See Mike S or Mat R).

Note that both of these values are displayed in "encrypted form" after they are entered.

 

Notes

This works only with EncounterPRO Client version 4.05.34 (or higher) and EncounterPRO Server version 4.05.2.8 (or higher).

JMJ Corportate office server is setup as a "clearing house" to receive the customer data and upload to customer's state registries thru a secured Webservice technology.
