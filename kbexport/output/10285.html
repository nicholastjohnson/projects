Error using Welch Allyn Vitals Monitor

10285

Introduction

When the user has a Welch Allyn Vitals Monitor then several items on the vitals screen should automatically be grey and filled in automatically from the device.

Symptom

When the vitals screen comes up, none of the fields are grey and there is an error message in the event viewer like this:

4.05.34.7 (05/19/2005) >>> u_component_observation_jmj_com - (xx_initialize()) Error calling com source (Error calling external object function initialize at line 55 in function xx_initialize of object u_component_observation_jmj_com.)

Cause

This is caused by incorrect versions of the Welch Allyn components of EproLibNet.  This can happen if previous versions of EpropLibNet installed these files in the program directory and the latest installation put these files in the common directory without removing them from the program directory.

Workaround

Delete the following files from the program directory:

WAVSMComSvr.dll

Interop.WAVSMCOMSVR_CESLib.dll

If the problem persists, rerun the EncounterPRO client installation and select the "Repair" option.
