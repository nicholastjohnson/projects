Upgrade procedures for version 4.0.04 to version 4.05.xx

10288

Introduction

This article will guide you through the upgrade of version 4.0.04 databases to the current version 4 database.

Symptom

Cause

Workaround

Resolution

 

 

Create an issue in Outlook and copy this document into the issue. While doing the upgrade if there are any unusual behaviors or problems with the upgrade, then document within the issue exactly what happen. If database is not “CPR” change all instances of CPR to database name

 

1)      Change RDP Settings - If there is an idle session limit set then change it to never but  remember what it was set to so you can set it back when the upgrade is done.  If you

2)      plan on disconnecting while doing the upgrade make sure you set the End a   

3)      disconnected session to never also.

4)      Backup of EncounterPRO Database using the backup file name Pre405Upgrade

5)      Using the Import Data in SQL feature create a new database and copy all tables except p_tables and name it cprpre405 and create Access link to database cprpre405

6)      Stop EncounterPRO Services and all instances of EncounterPRO, EPRO Server and EncounterPRO Service 

7)      Copy entire contents of EncounterPRO CD to the Hard Drive.

8)      If the Attachment Filegroup has not been created Click Here

9)      Open and run the Convert 4.0.04 to 4.05.sql

10)  Set the [customer_id] field in c_Database_Status with the below script.  Make sure you replace CUSTOMER# with the appropriate #.

UPDATE c_database_status
SET customer_id = ‘CUSTOMER#’

 

11)  Upgrade Mod Level to 109 Click Here through Query Analyzer

12)  Upgrade Database

 

·         Open the DB Maintenance Mode and click on “Updates” tab

·         Click on “Reset Remote Server” and verify “Remote Server Set” message pops up.

·         Click on “Check Remote Server” and verify that “Remote Server Available” message pops up.

·         Click on “Updates” tab

·         Click “Check Now”

·         On “Utility Scripts” tab, scroll down to “Sync c_Database_Table” and execute script

·         Upgrade “Database” to current mod level

·         Restart EPRO Server if “Reports” doesn’t appear yet

·         Upgrade “Reports

·         Run all Hotfixes

 

13)  Run DUMP TRANSACTION cpr WITH NO_LOG through Query Analyzer

14)  Backup Database and called it premidupgrade1.bak (optional)

15)  Open and run the Check Sync Status.sql script located on CD under EncounterPRO\Database\4.05\Scripts\Utility

16)  If the output says “This database has never been synced”, then open and run the All Sync Prep Scripts.sql

17)  Backup Database and called it premidupgrade2.bak (optional)

18)  Run the Check Sync Status.sql again.  Verify that it now says “Sync has already been performed on this database”

19)  Open the DB Maintenance Mode and click on “Updates” tab

20)  Open and run the Reset Identity Seeds.sql through Query Analyzer

21)  Upgrade “Config” through EPRO Server

22)  Check c_menu and c_workplan to make sure that there are the same amount of    records that are in the master

23)  Run DUMP TRANSACTION cpr WITH NO_LOG through Query Analyzer

24)  Open and run the Sync c_Age_Range_X through Query Analyzer

25)  Open and run the Convert 4.0.04 to 4.05 after sync.sql through Query Analyzer

26)  Open and run the Update o_server_component.sql through Query Analyzer

27)  Open and run the Sync c_Attachment_Extension Full Replacement.sql through Query Analyzer

28)  Open EPRO Server

29)  Run DB Scripts from Utility tab (for 126 click here)

30)  Run all Hot Fixes

31)  Run Modified Medication Repair Utility from Tools Tab on EPRO Server

32)  Run the stored procedure sp_rebuild_constraints ‘%

33)  Shrink DB in SQL to 5 %

34)  Delete old SQL Jobs and create SQL Jobs “4.0 Maintenance Jobs” (If they are a MPT practice consult with them first)

35)  If DB does not shrink Shrink Transaction Log to 1000 MB

36)  Backup Database and called it prefinalupgrade.bak (required) if space is limited overwrite initial backup and inform client

37)  Copy client files

a.      Create “EncounterPRO” directory under CPRFILES\Install if not already created

b.      Copy Install files to EncounterPRO directory

38)  If client is on Millbrook ensure that the application opens successfully. 

39)  RAR the pre-upgrade backup files

40)  Schedule a task in the Issues calendar, cc jmj-support, to delete the pre-upgrade backup one week after the upgrade
