Checkout Tab count is wrong

10290

Introduction

The Checkout Tab displays the encounters that were performed for a specified day.  The number listed on the checkout tab is the count of encounters finished today.

Symptom

The count displayed on the Checkout Tab is different than the actual number of encounters listed when the tab is selected.

Cause

The checkout screen displays the encounters for a specfied date and allows the user to change the date, change the office, or filter on billing status or patient name.  The number on the tab, on the other hand, is the count of the encounters finished today in the current office with no filters.  The discrepancy could be caused by the criteria on the checkout screen.  Note that the criteria may remain even when the users changes tabs or eve logs out and back in.

If the number on the tab is less than the number actually listed and there are no filtering criteria, then the problem could be a bug in the stored procedure that counts the encounters.  This bug fails to count encounters where the patient_location field is null. 

Workaround

First make sure that there are no filter criteria set on the checkout screen.

The bug in the stored procedure that counts the encounters is fixed in mod level 127 and in Hotfix 62

Resolution

Upgrade to mod level 127 or apply Hotfix 62.
