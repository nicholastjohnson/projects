Attachment Treatment folder workplans

10295

Introduction

Symptom

When posting an attachment to a treatment in a specific folder, that folder's workplan either sends a todo item or pops up a send todo service, and when the todo is viewed and the item box clicked on, the transcribe attachment service is launched, but it is the dashboard that is needed.

Cause

Workaround

Resolution

First, make sure that the folder workplan is a Treatment workplan

First, make sure that the folder workplan is a Treatmentworkplan.  If there is no Treatment workplan that has the right steps, createone.

  • Make the treatment type Lab (it doesn’t really matter what you pick, I don’t think)
  • Description: “Dashboard to (primary provider or whatever role)”
  • Add a step, edit the step, add service ‘Treatment Dashboard’
  • Leave “select observation” blank
  • Click on the item and pick Order For and pick the person / role / Primary provider
  • Edit Description and enter “View results for %WP%”
  • When you pick the Workplan Owner on the Attachment Workplan screen, pick “User” or “Any User” – this determines which user will be firing off the attachment workplan in the firstplace (the scanner)
  • Change Workplan Required to Yes if you want the workplan to fire automatically and you don’t want the user/scanner to pick which workplan is used.
