EproLibNet error running GRITS report

10298

Introduction

The article explains what to do when, while running the GRITS Batch report, you get an error that mentions EproLibNet.

This error should not happen particularly because the GRITS report should be run thru EproLib4, not EproLibNet

Symptom

An error like this:

Cause


Workaround

To troubleshoot:
1- Go to c_component_registry to verify that the GRITS report component is tied to EproLib4, not EproLibNet

If the GRITS report component is associated with EproLibNet, set it to EproLib4.
Elseif the GRITS report component is associated with EproLib4, verify thatthe practice is running the latest production build of the EncounterPROClient.


If the GRITS_REPORT row does not exist in c_component_registry, run this against the database:

INSERT INTO c_component_registry (component_id, component_type, component, description, component_class) VALUES ('GRITS_REPORT', 'REPORT', 'REPORT_ENGINE', 'GRITS Report', 'EPReport.GRITS')
