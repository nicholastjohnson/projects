Facility ID error when checking pts in

10299

Introduction

This article explains what to do if a practice that is integrated is not able to check in pts due to an error regarding an inccorect facility ID

Symptom

  1. Some aspect of the integration works. For example, pt demographics ill cross over.
  2. When a file is received from the PMand processed by an EncounterPRO service, something like the following shows up in the Event Viewer:
    1. u_hl7_arrived - (arrived()) SIU facility (2)not known =>.

Cause

The facility ID is not properly mapped in c_office table.

Workaround

check with the pratice and find out the list of facility id's in PM system. Map them in c_office table.

Resolution

  1. Go to c_component_attribute, filter for attribute=FACILITYID. Set the value for each row to the correct facility id.
  2. Go to c_office, verify that the billing_id (not the billing_code) is set to the facility_id
