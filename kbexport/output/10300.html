PM Integration Setup - HL7

10300

Introduction

This article explains how to setup a practice management integration with EncounterPRO.

This article assumes :

  1. There exists a practice that is presently an active customer of JMJTechnologies and EncounterPRO has been installed at the practice
  2. The EncounterPRO installation CD has been downloaded to the practice's server
  3. This type of integration is not in testing and is known to be HL7
    1. The testing status and HL7-ness of an integration type can be found here: file://///techserv/devel/Integration/PracticeManagement/Documents/Config/Setup%20PMS.htm
  4. The PM software is (or will be) installed on the same server as EncounterPRO

This article does not offer troubleshooting techniques for apre-existing PM installation, nor gives details on how to test theintegration after it has been set up.

Resolution

  1. Go to the install CD directories and go to Misc. Instructions\Integration Setup\HL7 Mapping
    1. If the integration is Centricity, open the Centricity folder and read the document in there
  2. How to select the reg file and bat file to edit:
    1. If this type of integration uses sockect connection transfers, edit the files
      • "DirectoryServerSocTx.reg" 
      • "HL7ConfigSocketTransfer.bat".
    2. Else if the integration is not Elligence, edit the files
      • "DirectoryServerFileTx.reg" 
      • "HL7ConfigFileTransfer.bat".
    3. If the integration is Elligence, open the Elligence folder and edit and run these files instead
  3. How to edit the reg file and bat files:
    • When editing the reg file,
      • If you are not using the D drive as your data drive, replace “d:\”
      • If setting up Elligence, replace #DATABASENAME# with the name of the database, minus the "dsi_" prefix.
      • Replace #DirectoryServer# and #ClientWorkstation# with the name of the server.
    • When editing the bat file,
      • Verify (by hand or by careful inspection) that all paths and directory changes will be valid.
      • If windows is installed at the server at C:\WINDOWS, replace "winnt" with "windows
  4. Run the bat file. This will automatically first run the regfile. Click OK when it asks if you want to import the registry changes.Pay close attention to the command prompt that follows to see if anyerrors occurred in the execution of the bat file.
  5. Go to EncounterPRO, Configuration, Practice Management, select the PM type.
    • If this type of PM is not listed, go to x_Integrations and add arow for this type of integration, similar to other HL7 integration type(like Centricity or Elligence).
    • For example: if the new integration type is WinMedStat, add a row with the following values:
      • Name= WinMedStat
        billing_system = WINMEDSTAT
        Schedule_Id = ENCOUNTERPRO_SCHED
        Billing_Id = ENCOUNTERPRO_BILL
        message_type_in = ENCOUNTERPRO_CHECKIN
        message_type_out = ENCOUNTERPRO_CHECKOUT
        message_in = TRN_IN_FILECOPY_HL7
        message_out = TRN_OUT_FILECOPY_HL7
        input_file = !dummy.dcom
        output_file = !dummy.dcom
        Enabled = Y
  6. From EncounterPRO, Configuration, Practice Management, enter the facility id, provider id, and appt type mapping.
    • If setting up Centricty, Elligence, or Compusense, this informationcan be found by going to Query Analyzer and running a script locatedhere:
      • Misc. Instructions\Integration Setup\Scripts\[Integration type]\List [Integration type] Facility & Provider Codes.sql
  7. Go to the table o_server_component and verify that allservices that should be active have status of OK and all services thatshould not be active have status=NA. Which services go with whichintegration type can be found here:
    • file://///techserv/devel/Integration/PracticeManagement/Documents/Config/Setup%20PMS.htm
  8. In c_component_attribute, set the facility id for all pertinent component_ids
