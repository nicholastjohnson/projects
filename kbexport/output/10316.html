Patient Import demographics not working

10316

Introduction

this article explains how the patient demographics are imported from the PM Systems

Symptom

a) new patient or updated patient info is not saved in EncounterPRO

b) patient files are queued up in the import path but it's not processed by EncounterPRO

Cause

Another possible reason why new patient records aren't created in EncounterPro is if there is an existing patient (in any status) that has the same billing id as the patient they're trying to check in. 

 

Workaround

Resolution

a) Check EPRO Interface services are started successfully.

b) Check the PM system is exporting the messages by looking in the EPRO import (shared) folder.For HL7 systems the import path is defined in windows registry under Hkey_Local_Machine->Software->AHC->Transport->EproServer. For non-HL7 systems it's in o_Message_Subscription table.

c) Files are queued up in import path but not picked up encounterpro:

It's possible that the service logon user (mostly eprojmj) is not given the local administrator privileges. OR

the folder is not given full permissions OR

the file extension exported ( HL7 / NON-HL7 look the defined extensions in their import path) is different from our mappings. for example in registry the file extension is given as ".txt" but exported is in ".dat or .hl7". these cases just change the file extension in registry to the exported file types for them to be read successfull.
