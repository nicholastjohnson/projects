Sent Message list takes too long to display

10317

Introduction

 

Symptom

When the user is on the Messages tab and selects the "Sent Messages" folder, EncounterPRO appears to freeze for a long time before the sent messages are displayed.

Cause

This is caused by EncounterPRO's default behavior of retrieving all sent messages for all time from the database.  When a customer has been using EncounterPRO for a long time this can be a substantial amount of data.

Workaround

There is a general preference called "000Standard Sent Items Display Date Range" or "Sent Items Display Date Range" which will limit the messages displayed to a period of time.  Valid units are "Year", "Month" and "Day".  Examples are:

3 Months
30 Days
1 Year

The shorter the time period, the less data so the faster the response.  At this point, however, EncounterPRO does not have an easy way to get back to earlier messages without resetting the preference.

Resolution

This has been identified as a problem and reported as Dev Issue # 6027.  It will be addressed in EncounterPRO Client version 4.05.35.
