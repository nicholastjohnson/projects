Error in trust relationship--EpIE upload gateway failure

10325

Introduction


Symptom

An error in the event log:
System - (CheckFinalStatus) The underlying connection was closed: Could not establish trust relationship with remote server.

Uploads to the EpIE Gateway do not work.

Cause

The Site certificate for JMJ Technologies is not trusted by default.  The certificate must be manually installed.

Workaround


Resolution

Perform the following steps on the computer experiencing the problem:

  1. If you are installing the certificate on a terminal server, run change user /install at the command prompt.
  2. Go to http://www.jmjtech.com/certsrv
  3. Click the link "Download a CA certificate, certificate chain, or CRL" at the bottom of the page.
  4. Click the link "install this CA certificate" at the top of the page.
  5. Click yes to acknowledge all warnings and install the certificate.
  6. If you are installing the certificate on a terminal server, run change user /execute at the command prompt.
