Prescription Sig gets cut off during printing

10330

Introduction

 

Symptom

A prescription contains all the necessary data in EncounterPRO and when the prescription report is "Previewed" it looks fine.  However, when the prescription report is sent to the printer some of the Sig information is cut off on the right side.

Cause

Some printers report their printing area and margin information to Windows differently than others, so there can be a slight descrepancy between printers when windows applications calculate when to wrap text to the next line.

Workaround

Add the following lines to c_Report_Attribute:

dbo_c_Report_Attribute

report_id

attribute_sequence

attribute

value

{ED21F61C-D7A2-40EF-9022-2FE20658839D}

1000095

dwmodify

signature.width=3154

{AD78CA36-789B-49F1-8C17-D76C730E488F}

1000095

dwmodify

signature.width=3154

 

Resolution

This problem is fixed in EncounterPRO Client version 4.05.35 and EncounterPRO Server version 1.2.5.
