Migrated Encounters don't show up on the Pick List for All Encounters

10342

Introduction

the article explains why some of the encounter types (may be a v2 or v4) does not show on the encounter types popup list in 'all Encounter' report.

Symptom

a) not showing all the encounter types used in encounter table.

b) version 2 encounter types not showing

Cause

a) may be the encounter type used in encounter table but disabled (status = 'NA') in c_encounter_type

b) the encounter type exists in encounter table but not in c_encounter_type (may happen with migrated customers because we are not migrating encounter types)

Workaround

This will be a problem only if they are filtering the encounters based on 'Encounter Type'. Yes, we do not migrate encounter types and I believe only this report allows encounters to be filtered based on encounter type. also the frequently used v2 encoutner types (like SICK / WELL / PHONE / TODO ) are already in version 4.

for a migrated customer: run the below query to see what encounter types are used in version 2 but missing in version 4

select distinct encounter_type from p_patient_encounter
where not exists
(
select * from c_encounter_type
where c_encounter_type.encounter_type=p_patient_encounter.encounter_type
)

if they want this report to include any of the above encouner types then I will recommend adding them in c_encounter_type with status of 'NA'. Even doing this will not show on the popup because we filter out only records with status='OK'. I wonder why we should filter based on status because if we used the encounter type for a while and later disabled we still need our reports to show list of used encounter types (in encounter table) no matter whether that's currently enabled or disabled. I can discuss this with to mark and make changes if necessary in our master database.. But in order for all the encounter types (used in encoutner table) to show on the popup change the query in c_component_param table as below.

For param_title = 'Select the desired encounter types'

change the query field column = select description,encounter_type from c_encounter_type where encounter_type in (Select distinct encounter_type from p_patient_encounter)
