Restoring a History Template in 2.0

Workaround


Resolution

To restore a history template in Version 2.0 the following needs to be done

  • Go to c_observation
  • Filter for correct observation_category (sometimes the practice might say it is a HPI but it turns out to be ROS)
  • Filter descending on Sort_sequence (The templates have a number in this field)
  • Restore the history that needs to be removed by setting the status to OK
  • If need be you can filter on the Description column
  • If this does not work request the practice for a patient example that had this history in the past as perhaps it is just part of a history that was removed.
