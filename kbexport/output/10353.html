Introduction

this article explains why a TODO item is not closed even though the user is successfully completing it..

Symptom

 

Version 2:

this happens only to "PA" user class. also this will happen only if they set the preference to send the lab results to encounter owner for review. you can find this preference record in o_preferences.

Cause

Version 2:

there is a bug in version 2 that keeps ordering a todo item (only for PA's) if the above preference is set even though the encounter owner is same as the reviewer.
