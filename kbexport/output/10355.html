Immunization Report locks up client

10355

Introduction

 

Symptom

When the user runs the immunization report (or a report containing a signature) Epro locks up at "Processing...Please wait" and had to ended via the task mgr.

An Error like this one appears in the event viewer:

Event Type: Error
Event Source: EncounterPRO
Event Category: None
Event ID: 0
Date:  8/22/2005
Time:  3:45:19 PM
User:  N/A
Computer: EPROSERV1
Description:
NWMP\eprojmj on EPROSERV1
4.05.32.27 (3/17/2004) >>> UNKNOWN CALLER - (SYSTEM ERROR) errornum:37
message :Type mismatch accessing external object property ink at line 11 in post_open event of object w_render_cic_signature.
where   :w_render_cic_signature
object  :w_render_cic_signature
event   :post_open
line    :11

 

Cause

This problem is caused by a NULL value being stored as a signature captured from a CIC signature capture device.  Prior to EncounterPRO Client 4.05.33, if the user clicks "Finished" on the signature capture screen without actually entering a signature, EncounterPRO would store a NULL for the signature.

Workaround

In the EncounterPRO Server DB Maintenance application, go to the "Utility Scripts" tab and run the "Remove Null CIC Signatures" utility script.

Resolution

This problem is fixed in EncounterPRO Client 4.05.33.
