Multiple user_ids or provider_ids in integration mapping

10358

Introduction

This article explains what to do if there is an integration setup that has more than one user_id or provider_id for one user.

Symptom

  • Practice has more than one instance of a user in Config,Users and they want to map one provider_id for this person (ex: issue#16505)
OR
  • Practice wants multiple provider_ids (from the mapping of the PM system) to apply to one user (ex: issue #20582)

Cause


Workaround

IF Practice has more than one instance of a user in Config,Users and they want to map one provider_id for this person
THEN a primary user_id must be chosen.
Multiple user_ids sharing a single provider_id is not alllowable at this time

ELSE IF Practice wants multiple provider_ids to apply to one user
THEN In table "B_provider_translation" add as many rows as needed for this complication:

Office_id=office_id (NOT billing_id) from c_office
External_id=provider_id from mapping
Epro_id=user_id from c_user
