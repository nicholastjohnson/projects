Importing portraits not working correctly

10359

Introduction

When the user clicks on the portrait box on the chart or the patient demographics screen, the Attachment List screen is displayed.  This screen is supposed to list all the portraits taken previously for this patient, and by clicking the "New Portrait" button the user can import a new portrait.

Symptom

Sometimes, particularly after an upgrade, the attachment list screen doesn't list previous portraits.  Also, client build 33 might start prompting the user for a folder.  For portraits, the folder selection should be automatic.

Cause

These behaviors are caused by a change in how EncounterPRO handles the placement of newly imported attachments into folders.

Workaround

Hotfix 88 adds logic to a trigger to make sure that any attachment imported into the portrait folder is automatically shown in the portrait box on the SOAP screen and on the demographics screen.  It will also re-connect any portraits which were imported into the Portraits folder already but were not given the title "Portrait".

After running Hotfix 88, the users no longer need to name the attachment "Portrait", even if they're using client version 33 or 34.

Resolution

This behavior is resolved in EncounterPRO Client 4.05.35.
