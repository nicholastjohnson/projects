Servicewill not start - Process.Kill

10362

Introduction


Symptom

EncounterPRO Service will not start.

Error in event log:

Service cannot be started. System.ComponentModel.Win32Exception: Access is denied
at System.Diagnostics.Process.Kill()

Cause

There are eproserver.exe processes running that the EncounterPROService is unable to kill.  The EncounterPRO Service will notstart as long as there are eproserver.exe processes running.  Thisis to prevent the same service item from running multiple times.

Workaround


Resolution

The eproserver.exe processes must be killed before the EncounterPROService will start.  If the processes cannot be killed throughtask manager, which is likely if the EncounterPRO Service was not ableto kill them, try using the NT Resource Kit tool kill.exe.

If kill.exe fails to kill the eproserver.exe processes, a computer restart will end them.
