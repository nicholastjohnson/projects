Comment results not saving

10366

Introduction

The ROS body system Yes/No screen has a yes/no format with a Perform results for "No," "Yes," and a comment result titled "add'l comment" at the end.  When the doctor enters a comment and then goes back to edit it, the previous entry is gone.

Symptom

Cause

If the comment result description contains an apostrophe, such as "add'll comment", it will not save. The character in the description is causing the problem.

Workaround

Resolution

Delete the comment result and create a new one without the apostrophe.

Notes

This is fixed in a later version of Config.
