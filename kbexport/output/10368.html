Customer database has the wrong customer_id

10368

Introduction

Symptom

The wrong customer_id is stored in c_Database_Status.  The customer_id from c_Database_Status is used to populate the owner_id field in many tables that allow users to edit their own configuration.

Cause

Workaround

Step 1 - Update the customer_id field in c_Database_Status to the correct value

Step 2 - Obtain the "Update owner_id.sql" script from the "JMJSupport\Epro DB Utility Scripts" folder on the JMJ FTP Site

Step 3 - If the incorrect customer_id was not 999, then edit the "Update owner_id.sql" by replacing the 999 with the incorrect customer_id that was in c_Database_Status

Step 4 - Execute the sql script in query analyzer.  Check for errors.

Resolution
