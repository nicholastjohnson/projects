Todo items not closing

10378

Introduction

Symptom

Extraneous todo items are generated and are not automatically closing when the corresponding encounter is closed or cancelled.

Cause

The extraneous todo items are left over when encounters are forced closed (using the Exit service) and EncounterPRO doesn't properly close workplan items in nested workplans.

Workaround

Hotfix 72 updates a stored procedure with the correct logic for forcing encounters closed.  It also runs several scripts to identify and close todo items that have already been erroneously left open.

Resolution

This problem is fixed in mod level 127.
