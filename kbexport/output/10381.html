History screen freezes

10381

Introduction

Symptom

When attempting to edit or create a new history, a popup window appears with the phrase "Expression is not valid".  Then the history screen comes up but EncounterPRO is frozen.

Cause

This condition is cause by EncounterPRO attempting to display a comment on the history screen where the comment definition (or comment-title) contains a single quote character.  EncounterPRO performs a search using the comment definition and the single quote character is used to delimit the paramters of the search.  When one of the parameters contains a single quote, EncounterPRO does not create a valid search string.

Workaround

Hotfix 81 and DB Mod Level 127 have added logic to the database to substitute a backwards quote for the single quote for all comment definitions.  This affects the records in p_Observation_Comment, c_Observation_Result, and, where appropriate, u_Top_20.

Do one of the following:

a) If the customer is between mod level 110 and 126, run Hotfix 81

OR

b) Upgrade the customer to mod level 127

Resolution

This has been reported as Issue # 6210 and will be fixed in a future release of EncounterPRO.
