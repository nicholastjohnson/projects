Service starts then stops

10382

Introduction

This article explains what to do if the EncounterPRO service does not stay started.

Symptom

1- Starting the epro service, immediately get this error:



2- In Event Viewer, no message referring to this.

3- Going to EPRO Server, click on "Set APP Role" and get a "Success Message" within a matter of seconds. It usually takes about 1-2 minutes

Cause

1- Mod level is not correct. Try upping the mod level to see if this is the cause.

2- User for the service is not correct. Try logging in to the server with the same user info as for this service to see if this is the cause.

3- Permissions not set correctly for cprsystems

There may be a problem with database permissions if you get an error like this:
SELECT permission denied on object 'o_Server_Component', database 'EncounterPRO', owner 'dbo'.
   at System.Data.SqlClient.SqlCommand.ExecuteReader(CommandBehavior cmdBehavior, RunBehavior runBehavior, Boolean returnStream)
   at System.Data.SqlClient.SqlCommand.ExecuteReader()
   at EncounterPROService.ServiceManager.GetSystemUserID(Int32 ComputerID)
   at EncounterPROService.ServiceManager.startServices()

Workaround

Try clearing the Event Viewer log or setting Event log to overwrite messages as necessary.
