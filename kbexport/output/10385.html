Unable to delete chart alert

10385

Introduction

Symptom

When the user attempts to delete a chart alert, the alert remains on the Chart Alert screen.  No error message is recorded.

In add to this, in version 2 it reports this error  "u_sqlca - (check()) SQL ERROR = (8115) Arithmetic overflow error converting IDENTITY to data type smallint. "

Cause

A trigger which handles the chart alert deletions is somehow removed from the database.

In to add the above if this is version 2 the identity column (alert_progress_sequence in p_chart_alert_progress)  data type is small int and if the identity count goes more 32767 then it throws "arithmetic overflow" error for further deletions.

Workaround

The trigger which handles Chart Alert deletions is re-added by Hotfix 83.  Alternatively, you can run the appropriate DB Scripts.sql script.

 

For version 2: ( To fix error  "u_sqlca - (check()) SQL ERROR = (8115) Arithmetic overflow error converting IDENTITY to data type smallint. " )

a) go to p_chart_alert_progress thru enterprise manager and drop it's triggers.

b) go to the table and change the column alert_progress_sequence data type from "smallint" to "int" for p_chart_alert_progress

c) run the hotfix 83 to re-add the triggers.

 

Resolution

The trigger will also be re-added to the database in Mod Level 127.
