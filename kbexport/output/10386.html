Changing the default process priority for EncounterPRO Agent Service

10386

Introduction


Symptom

Sometimes, processes spawned by EncounterPRO Agent (formerly EncounterPRO Server Service) may fail with bizarre and inexplicable errors.

Some versions of EncounterPRO Agent default spawned processes to priority 'Below Normal'.

You can check this in the task manager.  Right click a running EproServer.exe process and check the value selected in the 'Set Priority' sub-menu.  If the value is lower than 'Normal', try making the changes described in this article and see if the problems go away.

Cause

Processes running at 'Below Normal' priority may display unusual behavior if the system does not allocate enough processor time for the process to complete its tasks in a timely manner.

Workaround

Set the following values in the EncounterPRO database to change the process priority to 'Normal' for spawned tasks from EncounterPRO Agent.

c_Domain

domain_id

domain_sequence

domain_item

domain_item_description

EPROSERVERSERVICE

998

TaskProcessPriority

3

EPROSERVERSERVICE

999

ServiceProcessPriority

3


Resolution

This is fixed in EncounterPROService.exe 1.1.8.1, which has not been included in a setup package at the time of this article's creation.
