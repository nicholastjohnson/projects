Edited prescriptions not printing properly

10392

Introduction

Symptom

When the user edits a medication and prints it, the pre-edit version of the prescription prints out.

Cause

This condition is caused by the fact that when a user modifies a prescription, in the database a new treatment record is created with a new treatment_id.  This is done to preserve the original prescription data and show a correct history of when the prescription was originated, when it was changed, who changed it, etc.

Prior to build 33.15, the treatment dashboard would continue to display the original treatment even after the treatment was modified.  Any operations performed against the treatment, such as printing, would be performed against the original treatment until the user exited and reentered the dashboard, at which point the correct treatment would display.

 

Workaround

To get around this problem, the user may modify the treatment from the treatment dashboard, then exit and reenter the dashboard.  This causes EncounterPRO display the modified treatment and to perform any further operations, such as printing, against the modified treatment.

Resolution

There have been two resolutions to address different aspects of this problem. 

First, in build 33.15, the treatment dashboard was enhanced to recognize when the treatment was modified as a result of a button or hotsopt being clicked.  It would then display the data from the correct (moddified) treatment.

Second, in build 34, the prescription report was enhanced to recognize when it was being asked to print a treatment that had been modified.  This was done to catch the situation where a treatment is modified and printed in a single workplan before returning to the dashboard.
