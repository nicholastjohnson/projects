Modified prescription still shows original ordered-by provider

10393

Introduction

Symptom

When a user modifies a prescription treatment that was originally ordered by a different user, the original ordered-by user is kept as the ordered-by user for the modified prescription.

Cause

This problem is caused by the treatment modification logic preserving as many fields as possible from the original treatment definition.  In this case, the ordered-by user is overriding the current user in the creation of the modified treatment record.

Workaround

A workaround for this condition is provided in database mod level 127, or in mod level 126 with Hotfix 85 applied.

Resolution

This problem is fixed in EncounterPRO Client 4.05.35.
