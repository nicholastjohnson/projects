Room Menu displayed when performing a ToDo item

10406

Introduction

Symptom

When the user clicks on an item in his/her todo list, a room menu is displayed, even though the encounter is closed.

Cause

This will happen when an encounter is closed while there is still a room change pending for the patient.  This can be caused by a premature closing of the encounter or a misconfigured encounter workplan.

Workaround

Apply Hotfix 87.

Resolution

This issue is resolved in Database Mod Level 127.
