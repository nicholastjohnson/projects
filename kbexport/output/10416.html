Introduction

This article explains how to setup the billing component to export bills only once for a today at given time.

Notes

Currently Batch Billing only works if a practice sees less then 150 a day

Batch billing is useful if,

a) Customer makes lot of changes to chart after a patient is checked out.
b) Customer decided to receive all the bills at the end of the day

To setup a batch billing please complete the step 1 and step 2.

Step:1 Disable the real time billing

If you decided to setup billing in batch mode then 'Disable' the billing service in o_server_componentwith clear description as below so that no one will accidently enablethis. If this gets enabled then all the bills will get exported as soonas the encounter is closed.

Step:2 activate the batch billing

Setup a scheduled service in o_service_schedule table as below.

user_id = '#BILL'
Office_id=NULL
service='BATCH_BILLING'
schedule_type='DAILY'
schedule_interval = {enter the time when you want this batch to start ;for example 6:00 PM}
last_service_date={oneday lesser than batch activation date; for example if the batch needsto be activated from 12/1/2005 then this should be 11/30/2005}
created={date of creation}
created_by=SUPPORT
status ='OK

On the above configuration it starts processing all the queuedbilling requests at 6PM from 12/1/2005 onwards. Any billing requestsqueued up after 6PM will get processed only the next batch time (nextday 6PM).
