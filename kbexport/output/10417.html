Version 2.0

You cannot Merge Patients. Advise the practice to print out and scan in the information into the correct chart or to re-chart it and send us a chart of information to delete the incorrect chart.

Version 4.0 (Prior to Build 34 support has to run patient merges)

Verify all information first. Sometimes the Billing ID and CPR ID will be different. Check DOB and Patient Names before running scripts.

Run the following query when no one is using EncounterPRO

sp_patient_merge 'GOOD CPRID','BAD CPRID'
GO

If patient has more then one account run query as many times as needed by replacing the 'BAD ID. See below example.

sp_patient_merge 'GOOD ID','BAD ID 1'
GO

sp_patient_merge 'GOOD ID','BAD ID 2'
GO

Setting up the Merge Button in Build 34

  • Select Utilities
  • Select Menu Bar (usually called "Standard Utility Menu Default"
  • Select Edit Menu
  • Select New
  • Menu Item Type = Service
  • Menu Item = Merge Patients
  • Button Title = Merge Patients
  • Button Help = Merge Patients
  • Button = button02.bmp
  • Select Finished
  • Select OK
  • Select Finished
