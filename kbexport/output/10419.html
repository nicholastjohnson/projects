Report prints previews blank

10419

Introduction

Reports prints/previews blank. This is seen with migration sites and upgrades. Usual cause of problem is with c_report_attribute. This table provides the link between the c_Report_definition table and the c_display_script table.

Symptom

Reports prints/previews blank

Cause

Older versions of EncounterPro did not update the c_report_attribute table when user makes a new version of the report.

Workaround

Check c_display_script for different versions of the same report.

Customized versions display_script_id yusually start with counter set to 1000000.

Then go to c_report_definition and find the report name.

Copy the report_id for the report name in c_report_definition table to the clip board..

Go to c_Report_Attribute. Use the report_id from the clip board to find the entry in the c_Report_Attribute table. Change the display_script_id to the customized version.

Verify that this solved the problem.
