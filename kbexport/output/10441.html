Error msg when EncounterPRO Service is processing Lab Results

10441

Introduction

 

Symptom

4.05.2.9 (12/16/2005) >>> u_component_xml_handler_jmj - (interpret_xml_treatment) unable to find or create treatment

Cause

There are three reasons we might get the “treatment not found” error.  

The first is simply that the specific kind of lab hadn’t been ordered before.  We create the mapping between a lab code and a jmj code the first time a lab of that type gets manually posted.

The second is that the lab company sometimes has several different codes for the same kind of lab, and JMJ can have different observation_ids for the same kind of lab.  The differences can be based on patient demographics, billing info, specific sub-tests desired, or who knows what.  When the user orders the lab with a different observation_id or when the lab sends it back with a different lab id, the lab won’t match an existing treatment so there will be another one-time manual posting needed to create a mapping between the codes.

The third is when a lab gets ordered in the practice and gets sent to the lab but they forget to order the lab in EncounterPRO.

Workaround

Resolution

That is the standard message when Epro Server cannot automatically map the lab results to the right treatment.  It is issued by a scheduled service thread and can be ignored.
