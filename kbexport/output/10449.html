How to Specify Treatment List Ids for New Treatment and the New Composite Treatment buttons on the Order Treatment Screen

10449

Introduction

The Order Treatment Screen for an Assessment has two buttons that allow the user to order treatments during an encounter.  The buttons are called "New Treatment" and "New Composite Treatment."  This article describes how to specify the Treatment Type Lists that are associated with these buttons. 

This article applies to EPRO version 4.05.35 and later.  (Prior to EPRO version 4.05.33, the "New Treatment" button was linked to the Treatment Type List "SINGLE."  Prior to EPRO version 5.05.35, the "New Composite Treatment" button was linked to the Treatment Type List "!COMPOSITE.")

Symptom

A problem exists when the user selects the "New Treatment" or the "New Composite Treatment" button on an Order Treatment screen for an Assessment and the wrong treatment type options are displayed. 

Cause

The Treatment Type List that appears when a user selects the "New Treatment" or the "New Composite Treatment" button is controlled by four tables:  c_treatment_type, c_treatment_type_list, c_treatment_type_list_def, and o_service_attribute.   

All treatment types the user wishes to access from the Order Treatment Screen must be in the c_treatment_type table with an "OK" status.  Once the treatment type is included in the c_treatment_type table, it can be included in a Treatment Type List.  Treatment Type Lists are defined by treatment_list_ids in the c_treatment_type_list_def table and by groups of treatment_types in the c_treatment_type_list table. 

Once a Treatment Type List has been defined, the treatment_list_id can be linked to the "New Treatment" and the "New Composite Treatment" buttons by adding attributes to the ORDERTREATMENT service in the  in the o_service_attribute table. 

The attribute for ORDERTREATMENT that controls the list id for the "New Treatment" button is "single_treatment_list_id."

The attribute for ORDERTREATMENT that controls the list id for the "New Composite Treatment" button is "composite_treatment_list_id."

The value for each of these attributes is the treatment_list_id from c_treatment_type_list_def table. 

 

Workaround
