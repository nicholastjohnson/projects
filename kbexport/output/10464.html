Patient Attachment Duplicates

10464

Introduction

This article explains what to do with attachments in patients charts that are identical.

Symptom

There are two attachments in a patient's chart that have the same nameand content. Changing the name of one changes the name of the other.

Cause

The stored procedure that gets the list of patient attachments does nothave a conditional for the field in p_patient_progress for the field'current_flag='Y''

Workaround


Resolution

Hotfix 104

Notes

The code for this hotfix is the same as the code for the creation ofthe stored procedure sp_get_attachments, except it has this in thewhere clause of the IF section of the IF/ELSE:
     AND pp.current_flag='Y'
