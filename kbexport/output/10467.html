Edit-Refill prescriprion prints wrong provider signature

10467

Introduction

Symptom

When the user selects <Edit-Refill> on the treatment dashboard and after finishing the edits and the refill they print the prescription, the original provider still prints on the prescription.

Cause

The "Edit" part of the <Edit-Refill> workplan creates a new treatment and closes the old treatment with a status of "Modified".  The treatment dashboard knows how to display and work with the latest treatment, but since the <Edit-Refill> workplan never returns to the treatment dashboard before opening the refill screen, the refill progress note is actually applied to the original treatment and not to the new, modified treatment.  When the new modified treatment is then printed, the prescription report doesn't see the refill progress note so it reverts back to the original ordering provider.

Workaround

Apply Hotfix 106

Hotfix 106 updates a stored procedure which will make sure the Refill progress note gets applied to the correct treatment.  Hotfix 106 also repairs previous refills by re-applying them to the correct treatment.

Resolution

This problem is resolved in DB Mod level 127.
