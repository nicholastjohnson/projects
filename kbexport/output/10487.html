Onset and Duration not working

10487

Introduction

this article explains why the freeform history taking screen is not filling in the onset & duration  text automatically on the edit box

Symptom

Example: Chief Complaint (CC) screen when the user selects 3 & Weeks then clicking on set is not filling this text automatically on the edit box.

Cause

the print_flag might have disabled for the selected unit in c_unit table

Workaround

Resolution

Notes

When someone complains that clicking on the onset is not automatically filling in years / months/ weeks / days then it's possible that those UNIT entries in c_unit table is set with print_flag='N'. setting this to 'Y' for that unit record will fix this problem.
