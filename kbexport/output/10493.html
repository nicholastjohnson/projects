How to purge the recent selection list items

10493

Introduction

After using EncounterPRO for some time, the "Recent Selections" pick list on many screens can get unusably large.  This article explains how the purge recent selection list items.

Symptom

 

Cause

Each time a new value is entered into the freeform line, that value is added to the "Recent Selections" list, unless it is already on the list.  After some time, the number of items on the list can get so large that the list takes a noticable amount of time to load.

Workaround

Resolution

There are two preferences under "General Preferences" labeled "Purge Recent Picks" and "Purge Recent Picks Keep #". 

Purge Recent Picks - This preference controls whether or not the lists get purged every night.  The default is "No" (make sure you set this to 'YES' if you want the lists purged)

Purge Recent Picks Keep # - This preference controls how many list items to keep when the list gets purged back.  The default is 30.

These preferences added automatically by Mod Level 127.  If the database is on mod level 126, these preferences can be added by running Hotfix # 118.
