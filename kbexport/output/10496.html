How to fix the duplication of eproserver.exe duplicate services with EncounterPRO Service

10496

Introduction

This article explains how to fix the orphan "eproserver.exe" instances.  This fix should only be run on sites running encounterPRO server 5.1.4 or higher.

Symptom

a) check in's are duplicating

b) charges are duplicating

Cause

Service threads are not terminated while shutting down encounterproservice

Workaround

 

Resolution

a) Stop the EncounterPRO Service. Go to Task Manager and see whether any orphan "eproserver.exe" are still running. (If not that is fine.) If there are orphans running they should be killed automatically by installing the following.

b) Copy the EncounterPRO Server Program files (usu found on at D:\Program Files\JMJ\EncounterPRO Server); Paste and rename the folder OrgEncounterPRO Server. (we perform this step just in case we have to restore the original server files)

c) Go to Ftpsite and copy the files under EP_Integrations\Version4.0\EproService into the "EncounterPRO server" Program Files folder (when prompted, replace all three files)

d) Start the EncounterPRO Service. This should kill all the orphan threads before spawning any new instances.

e) Make sure all the services are working fine: 
        a) Launch Epro Agent Status (EncounterPROService Monitor)
        b) Verify services running equal amount of threads in task manager
        c) Stop and restart the services to make sure all threads successfully stop
