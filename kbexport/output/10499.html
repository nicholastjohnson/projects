Setting a graph as the default

10499

Introduction

This article explains how to set the graph that displays by default when the "Graph" chart tab is selected.

Symptom

Cause

Workaround

Resolution

What sets a graph as the default is whichever observation_id sorts first in the c_observation_Tree_root
Go to the c_observation_tree_root table.
Filter for root_category = GRAPH.
Sort ascending in the observation_id field.
Whichever observation_id is on the top is the default.

The only way to "set" one at the top is to change the observation_id from one to a "lower number" in all of the c_observation tables, most importantly

c_observation
c_observation_result
c_observation_tree
c_observation_tree_root

and refresh EncounterPro




Notes

To set the Hgt / Wgt/ BP graph as the default graph for the "Graph" chart tab, the following observation_id record must exist in c_observation table and in the c_observation_tree_root table:  !0^05hgtwgt

This observation_id will be added to existing sites when they are upgraded to Config Version 4.05.34 or higher.  The record in c_observation_tree_root may need to be added manually to sites installed prior to the release of Version 5.0.  If this record is manually added, the root category needs to be "GRAPH."  The sort_sequence should be '0."
