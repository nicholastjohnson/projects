unable to create new diagnosis or procedure if the ICD or CPT code already exists

10507

Introduction

 

Symptom

a) unable to create new diagnosis if the ICD9 code already exists.

b) unable to create new procedure if the CPT code already exists.

 

Cause

When a new diagnosis or procedure is created, EncounterPRO will check to see if a diagnosis or procedure with the same description and code already exists.  If so, then the existing diagnosis or procedure will be used instead of creating a new one.  These stored procedures were enhanced as part of improvements to the v2 to v4 Migratino process to include a check to see if the code alone already exists.  While this is the desire behavior for migrations, it is not the desired behavior for the EncounterPRO client.

Workaround

Hotfix # 65, which made the original migration enhancements, has been updated to include logic to allow the EncounterPRO client to create assessments and procedures where the icd or cpt already exists, as long as the description is different from the existing assessment or procedure.

Apply or Re-Apply Hotfix 65

Resolution

This problem is reported as DevIssue # 6839 and is permanently resolved as part of Mod Level 127.  The changes to the stored procedures may be applied to a Mod Level 126 database by applying Hotfix # 65.
