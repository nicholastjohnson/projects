How to check if the indexes are present on an individual table

10512

Introduction

When the performance of EncounterPRO deteriorates for some queries but not for others, it is possible that an index is missing on some table.  This article describes how to check a table to see if all of the indexes are present.

Symptom

Cause

Workaround

In the SQL Enterprise Manager, navigate to the database and table you want to inspect.

Right-click on the table, select All Tasks / Manage Indexes...

Do the same in a reference database that is on the same Mod Level (e.g. SpecialServ/cpr_405_production).

Visually inspect that the same indexes are on the table in both databases.

If they are not, then run sp_rebuild_constraints on that table.

 

Resolution

The DB Maintenance screen in Epro Server version 5.2 will include a utility script to check all tables for missing indexes and report the missing ones to the user.
