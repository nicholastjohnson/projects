Upgrade Procedures 2.21.??

 

 

       1.  Backup of EncounterPRO Database

 

       2. Stop EncounterPRO Services

 

       3. Copy entire contents of EncounterPRO CD to the Hard Drive.

a.       Donot remove the old EncounterPRO CD until the upgrade is complete and working.

b.      Ifthere is not already create a directory on the data drive called Install and thencreate a folder under Install called JMJ then place the copy of the CD in theJMJ directory using this format, CD2.21.xx (where xx = build #).

 

       4. Install EncounterPRO Client

a. Goto Cd2.21.44\EncounterPRO\scripts\2.21\client\44and run Setup.exe

a.       Ifterminal server you will need to put the session in Install mode.

                                                                    i.     Go to start run and type:  change user /install (or just install via“Add/Remove Programs”

 

       5.  Restart Server

a.   When the server comes back up you will get amessage saying that a service failed to start. It is the EncounterPRO service that is not starting because the databasehas not been upgraded yet.  Just ok themessage and continue with the upgrade process.

 

       6. Update Database

a.       Ifthis is an upgrade for build 32 or older you need to go into SQL Enterprisemanager and select the option Select Into/Bulk Copy.

b.      Runthe server manager and tell it yes when it asks if you want to upgrade.

c.       Oncethe upgrade scripts have run successfully the server manager should give you amessage of success.

d.      Ifyou get a message “Copy Scripts Failed” then you need to manually copy over thenew client install files to the install directory under  CPRFILES.

 

       7. Update Services Configuration to New Format

 

       8.  Setup Database Backup Configuration to NewFormat

 

       9. Check integration to make sure it is still working

 

       10. Walk customer through upgrading Fat Clients.

 

       11. Print release notes and inform customer to call support or training for anyadditional assistance.

a.   Create the following record in the c_domain table
