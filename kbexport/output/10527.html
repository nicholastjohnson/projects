Permission Denied

10527

Introduction

Symptom

Some aspect of EncounterPRO is behaving incorrectly and an error message like this one appears in the error log:

EXECUTE permission denied on object xxxxxxxx

 

Cause

To prevent unauthorized access to patients' confidential medical records, the EncounterPRO database has only the minimum permissions granted on the tables and stored procedures.  Furthermore, these permissions are granted to an application role and not a direct SQL user.  If any permission is missing for any reason, EncounterPRO will behave incorrectly and report "permission denied" SQL errors.

Workaround

The correct permissions on all EncounterPRO database objects are stored in a file on the FTP site.  Execute this file in the SQL Query Analyzer using an account that has DBO privileges.

EP_Quick_Fixes\Permissions CPRSystem 126.sql

Don't worry if the database is not on mod level 126 yet.  Regardless of the mod level, the query analyzer is likely to report a number of errors running this script like the one below.  These errors are OK and can be ignored.

Server: Msg 208, Level 16, State 11, Line 2
Invalid object name 'dbo.xxxxxxxxxxxx'.
