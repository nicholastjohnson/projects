Concurrent Use of the EncounterPRO Application by Multiple Users

10532

Introduction

This article discusses the following functionalities and protections: • The ability for multiple users to interact concurrently with the EncounterPRO application; • The ability for concurrent users to simultaneously view the same record; • The ability for concurrent users to view the same clinical documentation or template; and • the level of protection used to maintain the integrity of clinical data. "

Symptom

Cause

Workaround

Resolution

EncounterPRO allows multiple users to interact concurrently with the application.

EncounterPRO allows concurrent users to simultaneously view the same record.

 

EncounterPRO allows concurrent users to view the same clinical documentation or template.

The EncounterPRO database is hosted by Microsoft SQL Server.  Microsoft® SQL Server™ 2000 supports a wide range of optimistic and pessimistic concurrency control mechanisms.  EncounterPRO operates with the SQl Server default concurrency setting called "Read Committed".
