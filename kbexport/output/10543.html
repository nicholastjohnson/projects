Setting up Automatic File Import Epro version 4 and above

Introduction

EncounterPRO can automatically import files from specified directories using a combination of three EncounterPRO technologies:

 

  • The “File Import” External Source Component
  • The External Source Service
  • The Service Scheduler

To implement automatic file imports in EncounterPRO you will need to configure the Service Scheduler to run the External Source service at regular intervals, and further configure that External Source service to invoke the “File Import” external source plugin to get files and post them as desired into the EncounterPRO Database. Notes

EncounterPRO will use the filename_pattern to interpret the filename and locate properties that are embedded in the filename.  The following patterns are recognized:

 

%cpr_id%

%billing_id%

%encounter_id%

%problem_id%

%treatment_id%

%observation_sequence%

%attachment_type%

%folder%

“?” – matches any character

everything else is treated as a literal character to be found in the filename.

 

For example, if the filename_pattern is “xyz%cpr_id%_%folder%abc”, then encounterpro will expect to find files which begin with “xyz”, then everything up to the first underscore “_” will be interpreted as the cpr_id, then everything up to the first “abc” will be interpreted as the folder.  In this example, the filename “xyz12345.1_Patient Informationabc.bmp” will be interpreted such that the cpr_id is “12345.1” and the folder is “Patient Information”.
