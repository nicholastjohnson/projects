Ax Txs with NA'd observations or procedures

10546

Introduction

This is an ACCESS query (not SQL!) that finds NA'd observations that are still on the Assessment Treatment lists.  To fix this, replace all of the user_ids with "qzq(OriginalUserID)" so that they will never actually appear on anyone's Ax/Tx list, but it will be there for research purposes if needed later.

SELECT [dbo_u_assessment_treat_definition].[user_id], [dbo_u_assessment_treat_definition].[treatment_type], [dbo_u_assessment_treat_definition].[treatment_description], [dbo_u_assessment_treat_def_attrib].[attribute], [dbo_c_Observation].[description], [dbo_c_Observation].[status], [dbo_u_assessment_treat_definition].[assessment_id], [dbo_u_assessment_treat_definition].[definition_id]
FROM (dbo_u_assessment_treat_definition LEFT JOIN dbo_u_assessment_treat_def_attrib ON [dbo_u_assessment_treat_definition].[definition_id]=[dbo_u_assessment_treat_def_attrib].[definition_id]) LEFT JOIN dbo_c_Observation ON [dbo_u_assessment_treat_def_attrib].[value]=[dbo_c_Observation].[observation_id]
WHERE ((([dbo_u_assessment_treat_definition].[user_id]) Not Like "qzq*") And (([dbo_u_assessment_treat_def_attrib].[attribute])="observation_id") And (([dbo_c_Observation].[status])="na"));
