Migration Prep fails on Vaccine migration

10549

Introduction

Symptom

The Migration Prep fails on the Vaccine migration script with an error like this:

SQL ERROR = (8152) SQLSTATE = 22001
Microsoft OLE DB Provider for SQL Server
String or binary data would be truncated

Cause

This problem is caused by a vaccine description being longer than 40 characters.  During the migration process, drug definitions are created for each vaccine definition, and the maximum common name for a drug is 40 characters.

Workaround

Hotfix 127 adds logic to the trigger which creates the drug definition.  If the vaccine description is longer than 40 characters then it is truncated to 40 characters.

 

Apply Hotfix 127 and restart the Migration Prep from the "320 Vaccines.sql" script.

Resolution

This problem is permanently fixed in Mod Level 127.
