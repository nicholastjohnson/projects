Duplicate History Results After Migration

10554

Introduction

Symptom

After a Version 2 to Version 4 migration, the history screens (e.g. Review PFSH) show duplicate history results.

Cause

This can be caused when the migration scripts are re-run for any reason.  The scripts will usually detect data that has already been migrated and not re-migrate it, but in the case of histories the data is sometimes re-migrated producing duplicates.

Workaround

A utility script called "Fix v2 to v4 Duplicate Histories.sql" will detect and disable the duplicates.  Run the script from the Utility tab of the DBMaintenance program.

THIS UTILITY SCRIPT WILL LOCK UP THE DATABASE FOR UP TO 1 HOUR.  DO NOT RUN THIS UTILITY SCRIPT WHILE USERS ARE USING ENCOUNTERPRO.
