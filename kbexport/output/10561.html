Signature Capture Files

Introduction

 

At various points in EncounterPRO, the user is offered the opportunity to capture a hand-written signature.  The signature is always captured in the context of a clinical object such as an Encounter, a Medication Treatment, or Immunization Treatment.  Once captured, the information about the signature is encapsulated and stored in a file called a Signature Capture File.  The Signature Capture File is then attached to the clinical object which was the current context at the time of the capture.  For example, if the user is signing as part of a drug prescription refill service, the Signature Capture File is attached to the Medication Treatment which was being refilled.

 

The following types of Signature Capture Files may be found in EncounterPRO:

 

Penop_btoken

The first signature capture solution available with EncounterPRO, the Penop_btoken contains the signature data in a proprietary encrypted format.  In October, 2000, the PenOP company was acquired by Communication Intelligence Corporation, or CIC.

 

Cic_signature

Like penop, the CIC signature file contains the signature data in a proprietary encrypted format.  Outside of EncounterPRO, this format may only be read and displayed by software from CIC (www.cic.com).

 

Easyink_signature

The easyink_signature file format introduced in version 4 is created by a signature capture control from Odyssey Software, Inc.  The format is proprietary, but the ActiveX control is made freely available by Odyssey Software and Microsoft.

 

GIF

New with version 5, the JMJ Native Signature Capture component creates a standard image file in GIF format.  These files may be read and displayed by most computer systems.

 

 

 

 

How To Display Exported Signature Capture Files

 

In version 4, EncounterPRO introduced a Chart Export service which runs a set of reports and exports all attachments from a patient chart.  Most attachments are windows readable files which are easily displayed on most computer systems.  The Signature Capture Files, however, are not easily displayed outside of EncounterPRO.  For the penop_btoken and cic_signature files, software from CIC Inc. is needed to display the files.  For easyink_signature software could be written by anyone using the freely available easyink.ocx ActiveX control.

 

The easiest way to display a signature capture file is to log into the EncounterPRO system where it was captured, bring up the patient chart and navigate to the specific context object (encounter, treatment, etc) where the signature was captured.  The signature capture file will show as an attachment to the context object so simply click on the attachment and select “Display Attachment”.

 

If you have a signature capture file but do not have access to the original EncounterPRO installation, but you do have access to an EncounterPRO installation (a Demo installation will work), you can create a dummy patient and any treatment, and then import the signature capture file directly into the Attachments section of the treatment.  Then you can click on the attachment and select “Display Attachment”.
