version 2 signatures are not showing in encounter report

10578

Introduction

this article explains why signatures are not showing up in encounter report.

Symptom

chart->{patient}->audit shows that a signature is captured on that particular encounter  but it is ot showing in encounter reports.

Cause

attachment tag is not set as "Encounter".

Workaround

customers report that their signature are not showing in encounter reports. first run a audit to make sure sigature is captured for that encounter. if 'YES' then most probably the attachment tag is not set as 'Encounter' and so it is not displayed. to confirm this run the following queries.

a) the below query returns the list of encounter signatures logged with improper tags.

 select * from p_attchment_header where attachment_type='signature' and isnumeric(attachment_tag)=1

b)run the following query if there are any records returned fromt he previous query. this fixes the attachment tag issue and allows signatures to show up on the encounter report.

update p_attachment_header set attachment_tag='ENCOUNTER' where isnumeric(attachment_tag)=1 and attachment_type='signature'
