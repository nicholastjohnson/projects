Chart Export service does not appear to export all encounters

10584

Introduction

Symptom

When the chart export service is run, some of the encounters appear to be missing.  In particular, when there are multiple encounters on the same day, only one of the encounters for that day appears to have been exported.

Cause

This problem was caused by the naming convention used to save the exported encounter documents.  When the first encounter for a particular day was exported, the document was saved with the encounter date, not including the time, embedded in the document file name.  When the next encounter for the same date was exported, it would be given the same document name and would overwrite the first encounter document.

Workaround

There is a SQL script which will identify all of the Chart Export services performed where the patient had multiple encounters on the same day.  The output of this query can be used to identify patients and encounters that might have been missing from pervious chart exports.  Once identified, the desired encounter reports may be manually created and/or printed.

The script may be found here:

https://www.jmjtech.com/KB/support_scripts/version4/misc/Chart Exports Dup Encounter Date.sql

Resolution

This problem was resolved in EncounterPRO Client version 4.34.24.
