New charge is associating with too many assessments

10593

Introduction

When a new treatment charge is added to an encounter, it is supposed to automatically assciate iself with the same assessments that are associated with the treatment.

Symptom

A new treatment charge is added to an encounter and it is automatically associated with all assessments in the encounter, not just those associated with the treatment.

Cause

The "Add Charge" service is sometimes used to automatically or manually add a charge to a treatment.  This service calls an obsolete version of a stored procedure which does not include the correct logic for associating charges to assessments.

Workaround

To fix the problem prior to mod level 128, run Hotfix 139.

Resolution

This problem is resolved in Mod Level 128 and in EncounterPRO Client version 5.0.10.
