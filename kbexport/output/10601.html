A SQL error message indicates database corruption

10601

Introduction

Symptom

EncounterPRO reports a SQL error message which indicates an index or table may be corrupt.  Examples might be:

Warning: Fatal error 644 occurred at Oct 25 2006 11:47AM

Could not find the index entry for RID '36c494000000a2440504201a01e0323036302000' in index page (1:626553), index ID 0, database 'cpr'.

etc.

Cause

The database may have corrupted table data or index data.  This can be caused by unxpected process failure or hardware failure.

Workaround

Resolution

1)       Run “DBCC CheckDB” in the cpr database

2)       Search the output for all instances of the string “msg”.  These are error messages.

3)       For each table with error messages, run sp_rebuild_constraints
