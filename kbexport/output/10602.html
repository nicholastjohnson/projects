Setting Display Styles for Observations

10602

Introduction

The observation display style determines how the observation will be rendered on screens and in reports. 

Symptom

If  the description of the child observation is/isn't displaying, you can control what displays by using the display styles.

Cause

Workaround

A Quick How-To Lesson for Displaying or Not Displaying Comment Titles

 

If there is no value in the Display Style field on the Observation Configuration Screen, you can instruct that comment titles be displayed by entering the following:  OBO|RRR|CBT.  If you do not want comment titles to display, enter OBO|RRR|CBN.  If there is a value in the Display Style field, you can change only the comment display by changing only the last letter to “T” or “N.”  “T “ stands for Title.  “N” stands for No Title.

 

The Full Overview

 

If you want to learn more about Observation Display Styles, here are more complete instructions:

 

The observation display style determines how the observation will be rendered on screens and in reports.  The value consists of up to 3 groups of characters separated by the vertical bar character (|).  Each group of characters determines how one of the following aspects of the associated observation will be displayed:

 

1)                 Child Observations

2)                 Results

3)                 Comments

 

The first character of each group of characters determines which constituent to display.  The possible values are “O”, “R”, and “C” which represent Child Observations, Results, and Comments, respectively.  The second character in each group determines where the constituent will display relative to its parent, and the third character determines how to format the constituent.

 

The “where” character always has two possibilities:  “R” instructs EncounterPRO to display the constituent to the right of the observation description, and “B” instructs EncounterPRO to display the constituent below the observation description.

 

The possible “format” characters depend on which constituent is being displayed, but generally have three possibilities:  “R” = Display by result, “L” = Display by location, and “O” = Display by observation.

 

Each group may optionally specify three separators.  If specified, the separators immediately follow the “format” character and are delimited with the percent sign (%).  The first separator is the “Group” separator, the second separator is the “Title” separator, and the third separator is the “Item” separator.  The group separator is the characters which are inserted between sets of observation results.  The title separator is the characters inserted between the title of a set of observation results and the list of observation results.  The item separator is the set of characters inserted between individual results, if there is more than one result for the same observation.

When a composite or simple child is displayed below the parent observation, then the display style of the child observation will determine how it, in turn, is displayed.  When a composite or simple child is displayed to the right of the parent observation, then the display style of the parent determines how the child/children are displayed and the display styles of the children/grandchildren/etc are ignored.

 

The display style for an observation is stored in [c_Observation].[display_style].  If that field is null, then the display style will be taken from the associated [c_Observation_Type] record.  If there is no associated observation type, or the display style for the observation type is null, then the default display style is looked up in o_Preferences where the preference_type = ‘PREFERENCES’ and the preference_id = ‘default_display_style’.  If there is no default display style in o_Preferences, then the default display style will be:

 

“OBO; %: %, |RRR; %: %, |CBN; %: %, “

 

A fourth group may be added (also delimited with the vertical bar) with the first byte being “F”.  If the “F” group exists, then all characters following the “F” character will be interpreted as a formatting command according to the “Format Arguments” rules above.  For example:

 

“OBO; %: %, |RRR; %: %, |CBT; %: %, |Ffontsize=10,left,margin=0/2000/0“
