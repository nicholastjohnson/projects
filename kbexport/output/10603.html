Numeric results not displaying in drilldown template

10603

Introduction

This article discusses one possible cause of the problem of numeric results not displaying in a drilldown template.

Symptom

The problem occurs when a numeric result is entered by the user but the result does not display when the user moves up in the template.

Cause

One cause of the problem is a null display style for the parent of the observation in which the numeric result is entered.  For example, if a numeric result is entered within a simple observation called "HCT" and the observation "HCT" is the child of a parent observation called "CBC Results" and there is no display style set for "CBC Results," the numeric results of "HCT" will not display in the Lab template. 

Workaround

If the parent of an observation with numeric results is lacking a display style, edit the parent observation and add a display style.  A good default display style is "OBO|RRR|CRN."  (Be sure to use bars (shift + backslash) instead of slashes when you enter the display style as slashes will also cause display problems.)
