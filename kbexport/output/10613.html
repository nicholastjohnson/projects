EncounterPRO aborts without an error

10613

Introduction

Symptom

EncounterPRO aborts without any messages in the event viewer.  A popup message may appear when EncounterPRO aborts that says something like "unknown software exception ... occured in the application at location ..."

Cause

This condition may be caused by a user being set as their own supervisor on the User Definition screen.  Run the following query, it will detect the problem and fix it.  A list of the affected users who got their supervisor cleared will be displayed.  If no rows are returned, then that wasn't the problem.

DECLARE @loops TABLE (

      user_id varchar(24) NOT NULL,

      supervisor_user_id varchar(24) NULL,

      loop bit not null default (0))

 

INSERT INTO @loops (

      user_id,

      supervisor_user_id)

SELECT user_id,

      supervisor_user_id

FROM c_User

 

UPDATE x

SET loop = 1

FROM @loops x

WHERE x.user_id = x.supervisor_user_id

 

UPDATE x

SET loop = 1

FROM @loops x

      INNER JOIN c_User u

      ON x.supervisor_user_id = u.user_id

WHERE x.user_id = u.supervisor_user_id   

 

UPDATE x

SET loop = 1

FROM @loops x

      INNER JOIN c_User u1

      ON x.supervisor_user_id = u1.user_id

      INNER JOIN c_User u2

      ON u1.supervisor_user_id = u2.user_id

WHERE x.user_id = u2.supervisor_user_id  

 

UPDATE u

SET supervisor_user_id = NULL

FROM c_User u

      INNER JOIN @loops x

      ON u.user_id = x.user_id

WHERE x.loop <> 0

 

SELECT u1.user_id, u1.user_full_name, u1.supervisor_user_id, supervisor_full_name = u2.user_full_name

FROM @loops x

      INNER JOIN c_User u1

      ON x.user_id = u1.user_id

      INNER JOIn c_User u2

      ON x.supervisor_user_id = u2.user_id

WHERE x.loop <> 0
