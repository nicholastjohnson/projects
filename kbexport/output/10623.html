Auto Room Selection does not work with load balancing application servers

10623

Introduction

EncounterPRO allows the users to configure the room a particular computer is physically located in.  This configuration maps the computer id to a room id.  When load balancing is used, the actual server that a thin client connects to will vary from session to session.  Since the computer id for thin clients is a function of the application server and the windows user, the mapping from computer_id to room_id will only work if the application server selected for the windows session is the same as the application server used when the mapping was made.

Symptom

Cause

Workaround

There is no workaround.  Until a resolution is included in a future build of EncounterPRO, the auto-mapping of computers to rooms is unreliable in a load balancing environment.

Resolution

This issue has been identified as Dev Issue # 7238 and is scheduled for resolution in EncounterPRO version 5.
