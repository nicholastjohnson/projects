Protocol for Escalation to Product Development

10624

Introduction

Prior to escalating an issue to development, perform these steps, verifying that the problem still exists after each step.

 

  1. Search the knowledge base.  If similar issue is found then follow workaround/resolution instructions.
  2. Apply all hotfixes

 

If the problem still exists, then do the following:

 

  1. If not already done, personally reproduce the problem at the customer site
  2. If able to reproduce the problem at will at the customer site:
    1. Attempt to reproduce the problem in a local test database using the same client version and database mod level
    2. If the problem is reproduced locally, and a later version of Epro or the Mod Level is available, attempt to reproduce with the later version.  If later version fixes the problem, then schedule customer for upgrade.
    3. If the problem was not reproducable locally then schedule customer for a “Database Reset”  https://www.jmjtech.com/kb/Article.aspx?ArticleNumber=10622

 

 

If the problem still exists, then gather the following information:

 

  1. Client build and mod level
  2. Detailed instructions of how to reproduce.  Include whether reproducing in a local database was successful and which local database was used (even if not successful).
  3. All error messages (including warnings) that appear in the event viewer at the time of the anomalous behavior.  If none, please note that.
  4. If problem could not be reproduced locally, then include a SQL Trace of the problem:  https://www.jmjtech.com/kb/Article.aspx?ArticleNumber=10132
