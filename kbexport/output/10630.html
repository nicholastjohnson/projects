User not listed while searching from To TODO's

10630

Introduction

this article explains why the user is not showing in the search list from "To".

Symptom

user status is 'OK' but still not showing in the search..

Cause

Workaround

Resolution

The stored proc behind the screen called to retrieve the result is "sp_user_search" and this requests for info based on user status, status , role id, specialty id, name (search string), user class....

most of the times the status record was set to 'NA' which causes the record to be not shown in the list..

if every thing looks good then do a trace and find out the query (exec sp_user_search) that is posting to the database requesting this result.
