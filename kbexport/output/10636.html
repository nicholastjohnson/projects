Changing the Vaccine Consent Signature Statement

Cause

Workaround

Resolution

The default can be overridden from the interface in Configuration > Preferences:
Set the Global, General Preference "Vaccine Consent Statement" OR you can add a preference directly to the table 'p_preferences":
Preference_type = “PREFERENCES”
Preference_id = “Vaccine Consent Statement”
Unfortunately, it is currently limited to 255 characters.



You can display additional text by going directly into the tables. Add the following information into 'o_Service_Attribute'

  • service = VACCINE_SIGNATURE
  • attribute = Vaccine Consent Statement
  • value = TYPE YOUR TEXT HERE!

Notes

In config 19, mod level 137 the following logic is introduced: Added logic to look for "Vaccine Consent Statement" service attribute before preference. If attribute exists (up to 1024 characters) then it is used. If attribute does not exist, then preference is used.
