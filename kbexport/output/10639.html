How it works:

EncounterPRO will be periodically uploading all the vaccines (new or past immunes) that are not yet submitted to the registries thru a secured webservice to its Central IE (Integration Engine) Where it is then transformed to the format the different state registries understands and submits to them thru a secured transmission (SFTP / HTTPS etc)  mode. Currently, the interface transmitts the vaccine requests only once and it will re-submit if for some reason the vaccine data is updated / deleted after the transfer.
 
The current interface system assumes all the vaccines should be transferred to a single registry site. In case if your clinic is located in a state where it expects the data to transfer into multiple registry locations (based on county or zip or whatever like CA or NY) then we are not ready for your State Registry yet. Contact JMJ support to find out whether it has the electronic interface available for you.
 
Questions:
In case if you are looking to activate the state vaccine registry for your clinic then contact JMJ support and answer to the following questions.
 
a) do you want us to transfer all the vaccines data from day 1 of EncounterPRO? ( By default it starts from last 6 months)
b) do you need to get approval from every single patient before you transfer the data to state vaccine registry?
