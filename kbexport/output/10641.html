SECTION I Lab Integration Workplan/Configuration

Set up the default workplan for the Lab treatment type

In EncounterPRO, set default treatment workplan (the script should add the workplan, the trainer must assign as the default)

      While logged into practice's Epro icon go to > configuration > Treatment Type > Lab > edit treatment > click on the default workplan > do a search under description for “Integration” > specialty > choose the workplan for "Lab, Ongoing Clinical Staff Collect or Send/Refer Out (Integration)" > ok > finished.

 

Note: If this workplan does not appear, the practice may be on the wrong config level. Notify Support.

 

Default the Collect Specimen Menu

If the practice wishes to have the "Patient Referred To:" or "Specimen Sent To:" boxes in 'Collect Specimen Menu' default to a particular lab, see kb article #10640 "Default the name of a lab company to appear automatically in the "Collect or Refer Menu"

 

Assigning who receives the ToDo to Review the Lab Results

Once a lab has posted to a patients chart a TODO item will be sent to the user or role assigned in the workplan.  You need to assign a user or role the practice chooses to the last step in the workplan you created above.  Ask the practice who they prefer to receive "review lab" TODO item when a lab is posted.  A few common choices are as follows:

  1. Practice prefers the doctor who ordered the lab to receive the TODO then assign the role of primary provider.
  2. Practice wishes the encounter owner to receive the TODO then select the role of encounter owner.  Note: This could change frequently if any other kind of encounter is added after the lab is ordered.  ie telephone encounter.  To avoid this you can assign the role of workplan owner which will send the TODO to who created initial encounter.
  3. A popular option, especially for a big practice, is to have you create a new role titled "Lab Nurse" (or something similar). Remember to assign at least one appropriate user to the new role then that user can assign anyone else they choose.  To create a role see kb article #10119 "Adding Users".  Note: Not exactly accurate for adding 'roles' but best kb article I could find and works very similar.

To assign the user or role follow the below steps:

Click Config > workplan > workplan type > treatment > description > type “lab” > specialty > pick “Lab, ongoing clinical staff collect or send/refer out (Integration)” > display/edit > create local copy > edit workplan > review lab > edit step > encounter owner > ordered for > change the user to what the client would prefer. 

SECTION II Daily Lab Results

 

Adding a Daily Lab Results button to Utilities menu

To add the button to the Utilities menu, follow these steps:

Click Utilities > standard config menu default > create local copy (if not already there) > edit menu > new > service > general > referrals by day > finish > change button title to “Daily Lab Results” > finished > ok > finished > exit.

 

Fixing the "Referrals By Day" TODO bug

Note: This has yet to be approved by Devolopment but it does work

If practice is not on version 35.7 or higher and clicks on the "Daily Lab Results" button, it will always send a Not In Office TODO titled "Referrals By Day".  This can be fixed by doing the following:

  1. While logged into practice site, open the microsoft access db through Start > Programs > MS Access.
  2. Open the most current cpr.mdb practice has then open the dbo_o_service table and filter service column for "referrals by day" and change component id from svc_generic to SVC_UTILITY and then click mouse in a box below which will save work.

Note: NEVER CLICK "YES" WHEN ASKED IF YOU WANT TO SAVE CHANGES TO THE DESIGN OF TABLE
