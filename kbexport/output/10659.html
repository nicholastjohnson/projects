How to run a freeze trace for SQL 2005

10659

Introduction

This article provides information on how to run a trace in SQL 2005

Symptom

Cause

Workaround

 

Resolution

Open "SQL Server Profiler"

Select "New Trace":

General Tab:

*Enter trace name

 *Save to file

 *Set maximum size to 50 MB

(*If running a performance trace, Select "Enable trace stop time" and set to run to the end of the business day. )

 

Events tab:

 *Stored Procedures- RPC:Completed & Started

 *TSQL- SQL:BatchCompleted & BatchStarted

    To access all events if not shown by default, right click the deafult event and click "select event category". That should display all events available to edit. Then click "Deselect event category" and individually reselect each individual event you need to monitor.

 

Click the Column Filters button in the bottom right of the Events Selection tab.

 *For general performance issues, set a filter on the Duration: Equal to or greater than =1000 *For reproducable issues, set a filter on the SPID: Equal to (This can be found on the Server Status in VS 4.0 and Utilities Tab in VS 2.0)
