Set up an NFS Server in Windows

10667

Introduction

Medical Manager is a billing system that runs on UNIX.  Integrating with this billing system requires setting up an NFS (Network Filesystem) server on a Windows server.

Symptom

Cause

Workaround

Resolution

Download and install Windows Services for Unix 3.5 on the Windows server that will host the integration share.

Installation details:

  • choose custom installation
  • install only the following components:
    • NFS\Server for NFS
    • Authentication tools for NFS\User Name Mapping
    • Authentication tools for NFS\Server for NFS Authentication
Configuring the NFS server:
  • User Mapping
    • In the SFU folder (location where Services for Unix was installed, default is C:\SFU) create passwd.txt with the following content
      root:x:0:3:SuperUser:/:
    • In the SFU folder create group.txt with the following content
      root::0:root
    • Open Services for Unix Administration from the Start Menu
    • In "User Name Mapping" select the "Configuration" tab
      • Set mapping mode to "Use Password and Group files"
      • Set the location of the Password file (passwd.txt created above)
      • Set the location of the Group file (group.txt created above)
    • In "User Name Mapping" select the "Maps" tab
      • Ensure that "Simple maps" is not selected
      • Under "Advanced maps" click "Show User Maps"
      • Select the proper Windows domain name and click "List windows Users"
      • Select "eprojmj" from the list of Windows users
      • Click "List UNIX Users"
      • Select "root" from the list of UNIX users
      • Click "Add" to create the user map
    • Click "Apply" at the upper right corner of the window
    • Close Services for Unix Administration
Creating a share
  • Locate the directory that you want to share using NFS; right-click and select "Properties"
  • Click the "NFS Sharing" tab of the properties window
  • Select "Share this folder"
  • The "Share name" field is exactly what you use to mount the share from unix.  For example, if the Windows server's IP is 10.0.0.1 and "Share name" is "PMSHARE", the address of the NFS share is
    10.0.0.1:/PMSHARE
  • Leave the default values for "Encoding" and "Allow anonymous access"
  • Click the "Permissions" button
    • Change "Type of access" to "Read-Write"
    • Check "Allow root access"
    • Leave default value for "Encoding"
    • Click "OK" to close the permissions dialog
  • Click "OK" to save the NFS share settings
You have now configured an NFS host on the Windows server.
