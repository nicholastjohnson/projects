SQL Jobs Error 14274

10668

Introduction

Because MS SQL Server 2000 supports multiple instances, the originating_server field in the msdb..sysjobs table contains the instance name in the format ‘server\instance’. Even for the default server instance, the actual server name is used instead of (local). Problems then start arising when the actual Windows server is renamed and is called something different (which was the case in the testing environment). Any jobs therefore created before the renaming would have the old server name as a reference in the msdb..sysjobs table, and when trying to update/delete any of them, SQL Server would just throw a tantrum because the server names now do not match.

Symptom

Error 14274: Cannot add, update, or delete a job (or its steps or schedules) that originated from an MSX server. The job was not saved.

Cause

Server name does not match the referencing field (originating_server) in msdb..sysjobs table.

Workaround

  1. Update the originating_server field in the msdb..sysjobs table to reference the correct server name
  2. Update/Delete the Job
  3. Check back in the msdb..sysjobs to see if the delete worked, and hey presto, discover that all references to the job are gone.

Of course, the above workaround only took place after all possible backup measures had been implemented.
