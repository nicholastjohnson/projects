Linking tables in Access 2007

10669

Introduction

This article will provide step by step instructions to linking tables in Microsoft access to a Microsoft SQL database. or will it?.........

Symptom

Tables not being linked.

Cause

 

Workaround


Resolution



Open Microsoft Access.

Click on the "new blank database" icon

Name the database CPR
click "create"

When that opens, click the "External data" tab at the top

Click and drop down on the "More" menu item
select ODBC database

Select the "Link to the data source by..." radio button
click ok

Click "new", beside DSN name

Select the lowest option "SQL Server"

Name it CPR
click next
click finish

In this window, click the drop down the "Server" menu and select the name of the SQL server
click next

click next

On this window, select the "change the default database to" radio button,
Drop down and select the "CPR"  database
click next
click finish

Click "test data source"
if successful, click ok

Click ok on the Data source screen if "CPR" is available and highlighted

Click "select all"
click ok

At this point you may click "ok" on the first few popups, but i usually just hold the enter key and let them all fly by. Otherwise you may be clicking for quite some time.

When it finishes, the last table that should have been opened by default when the linking finished should be x_translation _set or something like it. You can start working by selecting the tables you need to work on to your left in the "all tables" section.
