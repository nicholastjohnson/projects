Resolution

Notes

a) check to make sure the directory has FULL (read & write) permissions on the folder where it is reading / writing the messages.

b) check in the o_message_subscription table to make sure the "address" field holds a valid path to import and export messages. It is important to check whether the file delimiters are entered properly. There are cases where it ignores the extension (say *.txt or *.* in case all files needs to be read) and in such case it is possible to get the "error logging incoming" message.
