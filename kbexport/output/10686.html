How to set up the NFS mount in SCO Unix for Medical Manager to EncounterPRO interface

10686

Introduction

These are instructions for properly configuring the mount to pmshare in SCO Unix, which is the operating system used on Medical Manager servers.

Symptom

Cause

Workaround

Resolution

Note: This guide assumes that the NFS share has already been hosted on the EncounterPRO server with XLink NFS Server. 

Information you must have to complete this guide:

  • IP Address of EncounterPRO server
  • IP Address of Medical Manager server
  • Name of NFS share on EncounterPRO server (e.g. /pmshare or /JMJ/PMSHARE)
    (the share name is case-sensitive)
  • Root password for Medical Manager server

  1. From a command prompt on the EncounterPRO server, telnet to the Medical Manager server
    example: telnet 10.0.2.198
  2. Enter the user and password when prompted.  The user name is root
  3. Change your terminal type to vt100 to avoid visual anomalies.
    TERM=vt100; export TERM
  4. Edit the /etc/default/filesys file
    (Notes about the     vi editor)
      • Move the cursor with the keyboard arrow keys
      • To edit, you must be in edit mode (hit I to enter edit mode)
      • To execute commands like :q, :wq, or :q! you must be in command mode (hit ESC to enter command mode)
      • If the display seems to be garbled or characters appear to be changing on their own, you may have the wrong TERM variable set; see Step #3
      • If you mess something up and want to start over, there is no undo.  To quit without saving, enter command mode (ESC if you are in edit mode) and type :q! then hit ENTER
    1. First, open the file
      vi /etc/default/filesys
    2. Using the arrow keys on your keyboard, move to the end of the file
    3. Hit I to enter edit mode
    4. Press ENTER to go to a new line
    5. Type the following where the cursor is (replace 10.0.2.199 with the IP Address of the EncounterPRO server and /jmj/pmshare with the name of the NFS share on the EncounterPRO server.  Use the TAB key to indent lines other than the first)
      bdev=10.0.2.199:/jmj/pmshare \
      mountdir=/usr2/jmjshare mount=yes \
      fstype=NFS desc="EncounterPRO PMShare" \
      mntopts=rw rcmount=yes
    6. Enter command mode by hitting ESC
    7. Type :wq and hit Enter to save changes
  5. Before we mount the NFS share, check to see whether the /usr2/jmjshare directory contains files
    ls /usr2/jmjshare
  6. If any files are listed, we must move them or the mount will fail.  If there are no files, proceed to the mount step (#8)
  7. Temporarily relocate the files in /usr2/jmjshare
    1. Create a new directory to hold the files
      mkdir /usr2/jmjshare_tmp
    2. Move the files to the new directory
      mv /usr2/jmjshare/* /usr2/jmjshare_tmp
  8. Mount the NFS share (replace 10.0.2.199 with the IP Address of the EncounterPRO server and /jmj/pmshare with the name of the NFS share on the EncounterPRO server)
    mount 10.0.2.199:/jmj/pmshare
  9. If files had to be moved from /jmj/pmshare before mounting, we have to move them back now
    1. Move the files back
      mv /usr2/jmjshare_tmp/* /usr2/jmjshare
    2. Remove the temporary directory
      rm /usr2/jmjshare_tmp
