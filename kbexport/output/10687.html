Adding a new SSL certificate to PHINMS 2.1 client cacerts

10687

Introduction

The PHINMS client needs SSL certificates (stored in C:\PHINMS2.1\client\config\cacerts) installed in order to function properly.  This "cacerts" file is a standard Java keystore and we will use the Java utility "keytool" to add a new certificate to the store.

Symptom

Cause

Workaround

Resolution

First, determine the location of your JRE or JDK.  For example, C:\Program Files\Java\jre1.6.0_06.
The keytool utility is located in the "bin" folder of your JRE/JDK folder.

Next, locate your SSL certificate file (*.cer format).  Decide on a name for the certificate; the web site the certificate is for will work well.  This name will be the alias under which the certificate is stored in cacerts.

Finally, locate cacerts, which by default is in C:\PHINMS2.1\client\config.

At a command prompt, run the keytool utility to add your SSL certificate to cacerts:

C:\>"<JRE/JDK Path>\bin\keytool.exe" -importcert -alias "<name for certificate>"  -file "<certificate path>" -keystore "<cacerts path>"

For example:
C:\>"C:\Program Files\Java\jre1.6.0_05\bin\keytool.exe" -importcert -alias www.inc.arkansasir.com -file "C:\Documents and Settings\chris\Desktop\inc_cert_new\inc_cert_new.cer" -keystore "C:\PHINMS2.1\client\config\cacerts"

Keytool will prompt for a password; the default password is "changeit".
